이 설명서는 DQMiner 의 효과적인 사용을 위해 단계별로 실습 형태의 시나리오를 제공하는 사용
가이드입니다. 본 문서를 진행하시기 전에 DQMiner 를 먼저 설치하시기 바랍니다. DQMiner 설치에
관한 정보가 필요한 경우, 별도로 제공된 설치 가이드를 참조하시기 바랍니다.

1부. 기본 분석
DQMiner 는 데이터의 품질을 체계적으로 진단 및 관리하기 위해 사용됩니다.
1 부에서는 DQMiner 사용을 위한 준비작업 및 사용법을 익히기 위한 간략한 사용법을 소개합니다.

1장. 준비  작업

1.1.  DQMINER  개요
DQMiner 개요

다양한 DBMS 에서 관리되고 있는 데이터 추출 및 분석을 위한 아키텍처를 채택하고 있으며 사용자는
웹브라우저를 통해 결과를 확인할 수 있도록 구성되어 있습니다.

1.2.  실습  환경  구성
이 실습을 위한 시스템 정보는 다음과 같습니다.
 OS: Windows 10
 Database: Oracle 12c

실습을 위해 제공되는 스크립트는 분석에 사용될 샘플 스키마와 데이터 생성을 위한 파일입니다.
실습을 위해 2 개의 DB 사용자(개발, 운영으로 가정)를 생성합니다.

아래의 절차를 따라 DB 사용자, 테이블, 샘플 데이터를 생성합니다.
1. https://github.com/gtoneqa/gtonesoft 로 접속하여 dq_sample 하위의  sql 파일을 다운로드 합니다.
2. sqlplus 또는 DB 관리툴을 통해 계정을 생성하고, sql 파일 내 스크립트를 실행합니다.

1) 커맨드창을 실행 합니다.
2) 커맨드창에서 sqlplus 실행 명령을 입력합니다.
예) C:\>sqlplus “/as sysdba”
3) 테이블스페이스를 생성 합니다.
예) SQL> create tablespace TS_DQ_TARGET_DEV DATAFILE ‘TS_DQ_TARGET_DEV.dbf' SIZE
100M AUTOEXTEND ON;

예) SQL> create tablespace TS_DQ_TARGET DATAFILE ‘TS_DQ_TARGET.dbf' SIZE 100M AUTO-
EXTEND ON;
4) DB 사용자 생성 및 기본 테이블스페이스 지정을 위한 명령어를 입력합니다.
예) SQL> create user C##DQ_TARGET_DEV identified by “DQ_TARGET_DEV” default tablespace
TS_DQ_TARGET_DEV;

예) SQL> create user C##DQ_TARGET identified by “DQ_TARGET” default tablespace TS_
DQ_TARGET;
 oracle 12c 이상에서는 일반사용자계정은 C##으로 시작하도록 규정하고 있습니다.
5) 생성한 DB 사용자에게 connect, resource, dictionary 권한을 부여합니다.
예) SQL> grant connect, resource, select any dictionary to C##DQ_TARGET_DEV;

예) SQL> grant connect, resource, select any dictionary to C##DQ_TARGET;
6) 일부 Oracle 버전은 기본 테이블스페이스 용량이 모두 사용되도록 허용하지 않습니다. 이런
경우, 테이블스페이스는 그 사용량에 대해 'unlimited'로 설정되어야 합니다.
예) SQL> ALTER USER C##DQ_TARGET_DEV DEFAULT TABLESPACE TS_DQ_TARGET_DEV
QUOTA UNLIMITED ON TS_DQ_TARGET_DEV;

SQL> ALTER USER C##DQ_TARGET DEFAULT TABLESPACE TS_DQ_TARGET QUOTA UNLIMITED
ON TS_DQ_TARGET;

7) 기존 연결을 종료하고 1 번째로 생성한 DB 사용자로 로그인 하여 github 에서 다운로드한
sql 파일내용을 복사하거나 파일을 특정 위치에 두고 실행합니다.
예) SQL> exit
예) C:\>sqlplus “C##DQ_TARGET_DEV/DQ_TARGET_DEV”
예) SQL> @DQ_TARGET_DEV.sql
8) 기존 연결을 종료하고 2 번째로 생성한 DB 사용자로 로그인하여 github 에서 다운로드한
파일을 실행합니다.
예) SQL> exit
예) C:\>sqlplus “C##DQ_TARGET/DQ_TARGET”
예) SQL> @DQ_TARGET.sql

2장. 진단 대상  등록
이번 장에서는 데이터 품질 진단을 위한 대상을 등록하고 권한을 설정하는 실습 사례를 제공합니다.

실습을 수행하기 앞서 웹 페이지 접속 방법은 아래와 같습니다.
1. 웹 브라우저를 실행합니다.
2. DQMiner URL(예: http://127.0.0.1:9095)로 접속합니다.
3. 관리자 용 ID 와 패스워드를 입력합니다(기본 값 DQMiner / DQMiner)
4. [Login] 버튼을 클릭합니다.

2.1.  진단대상  등록
진단 대상 스키마의 메타정보와 데이터를 수집하기 위해 데이터베이스에 연결하고 진단대상을
등록합니다.
1. ‘진단 대상 관리’ 메뉴를 클릭합니다.
2. [시스템 추가] 버튼을 클릭합니다.
3. 시스템 정보를 입력하고 [확인] 버튼을 클릭합니다.
항목 예시(추천)
*시스템 명 운영-DQ_TARGET
설명 데이터 품질 진단 대상 시스템 – 운영계

4. 추가한 시스템을 선택하고 [데이터베이스 추가] 버튼을 클릭합니다.
5. 데이터베이스 기본정보를 입력하고 [확인] 버튼을 클릭합니다.
항목 예시(추천)
*데이터베이스 명 운영 DB
설명 데이터 품질 진단 대상 데이터베이스 – 운영계

6. 추가한 데이터베이스를 선택하여 “데이터베이스” 탭에 연결정보를 입력하고[연결테스트]버튼을
클릭합니다.
항목 예시(추천)
*연결명 DQ_TARGET
*유형 ORACLE

2.2.  업무  설정
DQMiner 는 권한 관리를 위해 업무를 활용합니다. 업무를 생성하여 진단대상 테이블을 매핑합니다.
1. '진단 기준 관리' > ‘업무’ 메뉴를 선택합니다.
2. [선택된 업무 하위에 업무 추가] 버튼을 클릭합니다.
3. 업무명을 입력하고 [확인] 버튼을 클릭합니다.
항목 예시(추천)
*업무 명 운영-데이터품질

4. 추가한 업무를 선택하고 “진단 대상 테이블” 탭에서 [진단 대상 테이블을 추가합니다.] 버튼을
클릭하면 테이블 선택 창이 나타납니다.
5. “테이블 선택” 창에서 “C##DQ_TARGET"스키마를 선택하고 [검색] 버튼을 클릭합니다.  검색된 모든
테이블을 선택하고 [확인] 버튼을 클릭합니다.
6. “진단 대상 테이블”에 테이블이 추가된 것을 확인하고 [저장] 버튼을 클릭합니다.

3장. 사용자  권한  설정
사용자/그룹/역할 정보를 활용하여 업무권한, 메뉴권한, 보고서 권한 관리를 할 수 있습니다.
3.1.  역할  설정
DQMiner 의 권한 제어를 위해 역할을 생성하고, 권한을 부여하는 기능입니다. 설치 시 기본으로 5 개(IT
담당자, 업무 담당자, 품질 관리자, 품질 담당자, 협업 관리자)의 역할이 생성되며, 각 역할에 맞는 권한이
설정되어 있습니다.

이번 실습에서는 “Developer”역할을 생성하고 개발자가 주로 사용하는 기능에 대해서만 권한을
부여합니다.

역할 추가를 위해서는,
1. ‘사용자/권한’ > ‘역할’ 메뉴를 선택합니다.
2. [추가] 버튼을 클릭하여 역할 추가 창에서 역할명과 설명을 입력하고 [확인] 버튼을 클릭합니다.
항목 예시(추천)
* 역할 명 Developer
설명 IT Develope

추가한 역할에 메뉴 권한을 부여하기 위해서는,
1. 추가한 역할을 클릭합니다.
2. 메뉴 권한 탭에서 해당 역할에서 사용할 메뉴들을 체크하고 [저장] 버튼을 클릭합니다.

항목 예시(추천)
* 역할 명 Developer
메뉴 진단 실행, 진단 결과, 작업 로그, 스케줄, 보고서, 협업

4장. 분석 실행
분석 실행 실습을 수행하기 앞서 권한이 설정된 아이디로 로그인 하는 방법은 아래와 같습니다.
1. 우측 상단의 DQMiner 를 클릭하고 [로그아웃]을 클릭합니다.
2. DQMiner URL(예: http://127.0.0.1:9095)로 접속합니다.

4.1.  컬럼  분석  (통계분석 , 빈도분석 , 날짜분석 )
데이터 프로파일링을 위해 컬럼 분석을 사용합니다. 컬럼 분석은 데이터의 유형과 성격에 따라
분석유형을 설정하고 유효 값을 관리할 수 있는 기능입니다.

통계분석, 빈도분석을 수행하기 위해서는,
1. ‘진단 실행’ > '컬럼 분석' 메뉴를 선택합니다.
2. 상단 검색조건에서 업무, 데이터베이스를 선택하고 테이블 명을 입력하고 [컬럼 검색]버튼을
클릭합니다. 테이블명 입력 시 ‘%’문자로 LIKE 검색을 사용할 수 있습니다.
항목 예시(추천)
*업무 운영-데이터 품질
*데이터베이스 전체 데이터베이스
*테이블 검색 키워드 %user

Q1: DQMiner 웹에 어떻게 로그인하나요?  
A: DQMiner 웹에 로그인하려면 먼저 웹 브라우저를 열고 DQMiner 웹사이트 주소를 입력합니다. 페이지가 열리면 오른쪽 상단에 있는 '로그인' 버튼을 클릭합니다. 이후 사용자 ID와 비밀번호를 입력한 후 '로그인' 버튼을 클릭하여 시스템에 접속합니다.

Q2: DQMiner에서 데이터를 어떻게 다운로드하나요?  
A: 데이터를 다운로드하려면 DQMiner 웹에 로그인한 후, 왼쪽 메뉴에서 '데이터 관리'를 클릭합니다. 원하는 데이터 세트를 선택한 후, 오른쪽 상단에 있는 '다운로드' 버튼을 클릭합니다. 파일 형식을 선택하고 '확인'을 클릭하면 다운로드가 시작됩니다.

Q3: DQMiner에서 특정 데이터를 검색하려면 어떻게 하나요?  
A: DQMiner 웹에 로그인한 후, 상단 메뉴에서 '데이터 검색'을 선택합니다. 검색창에 원하는 키워드를 입력하고 '검색' 버튼을 클릭합니다. 검색 결과에서 원하는 데이터를 클릭하여 상세 정보를 확인할 수 있습니다.

Q4: 계정 정보를 어떻게 수정하나요?  
A: 계정 정보를 수정하려면 DQMiner 웹에 로그인한 후, 오른쪽 상단의 사용자 아이콘을 클릭합니다. '계정 설정'을 선택한 후, 수정하고 싶은 항목을 변경하고 '저장' 버튼을 클릭하여 변경사항을 저장합니다.

Q5: DQMiner 웹에서 지원을 요청하려면 어떻게 해야 하나요?  
A: DQMiner 웹에 로그인한 후, 하단 메뉴에서 '고객 지원'을 클릭합니다. 지원 요청 양식이 나타나면 필요한 정보를 입력하고 '제출' 버튼을 클릭하여 지원 요청을 전송합니다.

Q16: DQMiner의 개요는 무엇인가요?  
A DQMiner는 데이터 품질 분석 및 개선을 위한 도구로, 데이터의 정확성, 완전성, 일관성 등을 평가하는 기능을 제공합니다. 사용자는 이 도구를 통해 데이터 분석을 수행하고, 데이터 품질 문제를 식별하여 해결할 수 있습니다. DQMiner를 사용하기 위해서는 먼저 소프트웨어를 설치하고, 실습 환경을 구성해야 합니다.

Q17: DQMiner를 사용하기 위한 실습 환경은 어떻게 구성하나요?  
A 실습 환경을 구성하기 위해서는 다음 단계를 따르세요. 먼저 DQMiner 소프트웨어를 다운로드하여 설치합니다. 설치 후, 프로그램을 실행하고 초기 설정을 진행합니다. 이후 데이터베이스 연결을 설정하고, 분석할 데이터를 준비합니다. 이 과정에서 필요한 데이터 형식과 샘플 데이터를 확인하여 환경을 맞추는 것이 중요합니다.

Q18: 진단대상 등록은 어떻게 하나요?  
A 진단대상을 등록하려면 DQMiner 프로그램을 실행한 후, 상단 메뉴에서 '진단대상'을 클릭합니다. 그 다음 '등록' 버튼을 선택하여 새로운 진단대상을 추가합니다. 필요한 정보를 입력하고, 데이터 소스와 연결을 설정한 후 '저장'을 클릭하여 등록을 완료합니다. 등록된 진단대상은 이후 분석 작업에 활용됩니다.

Q25: 업무 설정을 어떻게 할 수 있나요?  
A: 업무 설정을 하려면 먼저 웹 화면의 왼쪽 메뉴에서 '업무 설정' 항목을 클릭하세요. 이후 나타나는 화면에서 필요한 업무 항목을 추가하거나 수정할 수 있습니다. 각 항목 옆에 있는 '편집' 버튼을 클릭하여 세부 설정을 조정한 후, '저장' 버튼을 눌러 변경 사항을 적용하세요.

Q26: 사용자 권한 설정은 어떻게 하나요?  
A: 사용자 권한 설정을 하려면 왼쪽 메뉴에서 '사용자 권한 설정'을 선택하세요. 여기서 '역할 설정', '그룹 설정', '사용자 관리'의 세 가지 옵션이 있습니다. 각각의 옵션을 클릭하여 원하는 권한을 설정하고, 변경 사항을 저장하세요.

Q27: 역할 설정은 어떻게 진행하나요?  
A: 역할 설정을 하려면 '사용자 권한 설정' 메뉴에서 '역할 설정'을 클릭하세요. 이후 '새 역할 추가' 버튼을 클릭하여 새로운 역할을 생성하고, 해당 역할에 부여할 권한을 체크한 후 '저장' 버튼을 눌러 완료하세요.

Q28: 그룹 설정은 어떻게 하나요?  
A: 그룹 설정을 하려면 '사용자 권한 설정' 메뉴에서 '그룹 설정'을 선택하세요. '새 그룹 추가' 버튼을 클릭하여 그룹 이름을 입력하고, 해당 그룹에 포함할 사용자를 선택한 후 '저장' 버튼을 눌러 그룹을 생성하세요.

Q29: 사용자 관리는 어떻게 하나요?  
A: 사용자 관리를 하려면 '사용자 권한 설정' 메뉴에서 '사용자 관리'를 클릭하세요. 여기서 기존 사용자의 목록이 나타납니다. 사용자를 추가하려면 '새 사용자 추가' 버튼을 클릭하고, 필요한 정보를 입력한 후 '저장' 버튼을 눌러 사용자 등록을 완료하세요.

Q30: 분석 실행은 어떻게 하나요?  
A: 분석 실행을 하려면 웹 화면의 왼쪽 메뉴에서 '분석 실행'을 선택하세요. 이후 분석할 데이터를 선택하고, 필요한 분석 옵션을 설정한 후 '실행' 버튼을 클릭하면 분석이 시작됩니다. 분석 결과는 지정된 화면에서 확인할 수 있습니다.

Q31: 컬럼 분석을 어떻게 시작하나요?  
A 컬럼 분석을 시작하려면, 먼저 분석하고자 하는 데이터셋을 열어야 합니다. 상단 메뉴에서 '분석' 탭을 클릭한 후 '컬럼 분석' 옵션을 선택하세요. 이후 통계분석, 빈도분석, 날짜분석 중 원하는 분석 유형을 선택하고, 분석할 컬럼을 지정한 후 '실행' 버튼을 눌러 결과를 확인할 수 있습니다.

Q32: 코드 분석에서 공통코드를 어떻게 확인하나요?  
A 코드 분석을 위해서는 먼저 '분석' 메뉴로 이동한 후 '코드 분석'을 선택하세요. 그 다음 '공통코드' 옵션을 클릭하면, 시스템에서 제공하는 공통코드 목록이 나타납니다. 여기서 원하는 코드를 선택하여 상세 정보를 확인하거나, 필요한 경우 새로운 코드를 추가할 수 있습니다.

Q33: 관계 분석에서 참조무결성을 어떻게 검토하나요?  
A 관계 분석을 수행하려면, '분석' 메뉴에서 '관계 분석'을 선택한 후 '참조무결성' 옵션을 클릭하세요. 그러면 데이터 간의 참조 관계가 시각적으로 표시됩니다. 각 관계를 클릭하여 상세 정보를 확인하고, 문제가 있는 경우 수정할 수 있는 옵션도 제공됩니다.

Q34: 업무 규칙을 설정하려면 어떻게 해야 하나요?  
A 업무 규칙을 설정하기 위해서는 '설정' 메뉴로 가서 '업무 규칙' 섹션을 선택하세요. 여기에서 '새 규칙 추가' 버튼을 클릭하고, 규칙의 이름과 조건을 입력한 후 '저장' 버튼을 눌러 규칙을 적용할 수 있습니다. 필요시 기존 규칙을 수정하거나 삭제할 수도 있습니다.

Q35: 스케줄 설정은 어떻게 하나요?  
A 스케줄 설정을 하려면, '설정' 메뉴에서 '스케줄 설정' 옵션을 선택하세요. 이후 '새 스케줄 추가' 버튼을 클릭하고, 스케줄의 이름, 주기, 시작 날짜 및 시간을 입력한 후 '저장' 버튼을 눌러 스케줄을 등록합니다. 등록된 스케줄은 목록에서 확인하고 수정할 수 있습니다.

Q36: 진단대상 등록은 어떻게 하나요?  
A: 진단대상을 등록하려면 먼저 웹 애플리케이션에 로그인한 후, 상단 메뉴에서 '진단대상' 탭을 클릭합니다. 그 후 '등록' 버튼을 찾아 클릭하여 진단대상 등록 양식을 엽니다. 필요한 정보를 입력한 후, '저장' 버튼을 눌러 등록을 완료합니다.

Q37: 업무 설정은 어떻게 진행하나요?  
A: 업무 설정을 하려면 로그인 후, 왼쪽 사이드바에서 '업무 관리' 메뉴를 선택합니다. 그 다음 '업무 설정' 옵션을 클릭하여 설정할 업무를 선택합니다. 필요한 세부 사항을 입력한 후, '적용' 버튼을 눌러 변경사항을 저장합니다.

Q38: 업무 권한 설정은 어떻게 하나요?  
A: 업무 권한 설정을 위해 먼저 로그인 후, '사용자 관리' 메뉴를 클릭합니다. 이후 권한을 설정할 사용자를 선택하고 '권한 설정' 버튼을 클릭합니다. 원하는 권한을 체크한 후, '저장' 버튼을 눌러 설정을 완료합니다.

Q39: ctq란 무엇이며 어떻게 관리하나요?  
A: ctq는 'Critical to Quality'의 약자로, 품질에 중요한 요소를 의미합니다. ctq를 관리하려면, 웹 화면에서 '진단기준관리' 메뉴를 클릭한 후 'ctq' 항목을 선택하세요. 여기에서 ctq 항목을 추가, 수정 또는 삭제할 수 있으며, 각 ctq의 중요도와 관련된 세부 정보를 입력할 수 있습니다.

Q40: dqi는 무엇을 의미하며 어떻게 활용하나요?  
A: dqi는 'Data Quality Indicator'의 약자로, 데이터 품질을 평가하는 지표입니다. dqi를 활용하려면 '진단기준관리' 메뉴에서 'dqi' 항목을 선택하세요. 이곳에서 각 데이터 항목의 품질 지표를 설정하고, 필요한 경우 수정하여 데이터의 신뢰성을 높일 수 있습니다.

Q41: 사용자 정의 코드를 어떻게 생성하나요?  
A: 사용자 정의 코드를 생성하려면, 웹 화면에서 '진단기준관리' 메뉴를 클릭한 후 '사용자 정의 코드' 항목으로 이동하세요. 여기에서 '새 코드 추가' 버튼을 클릭하고, 코드의 이름과 설명을 입력한 후 '저장' 버튼을 눌러 생성할 수 있습니다.

Q42: 사용자 정의 분석 기능은 어떻게 사용하나요?  
A: 사용자 정의 분석 기능을 사용하려면, '진단기준관리' 메뉴에서 '사용자 정의 분석' 항목을 선택하세요. 이후 '새 분석 생성' 버튼을 클릭하고, 분석의 기준과 조건을 설정한 후 '실행' 버튼을 눌러 분석을 시작할 수 있습니다.

Q43: 진단부가항목은 어떻게 추가하나요?  
A: 진단부가항목을 추가하려면, 웹 화면에서 '진단부가항목' 메뉴를 클릭하세요. 그 다음 '새 항목 추가' 버튼을 클릭하고, 항목의 이름과 설명을 입력한 후 '저장' 버튼을 눌러 추가할 수 있습니다.

Q46: 데이터원천에 대한 정보는 어디에서 찾을 수 있나요?  
A: 데이터원천에 대한 정보는 7.1 항목에서 확인할 수 있습니다. 이 항목을 찾으려면 메뉴에서 '데이터원천' 섹션을 클릭한 후, 해당 페이지에서 필요한 정보를 자세히 읽어보세요.

Q47: SQL 변수를 사용하는 방법은 무엇인가요?  
A: SQL 변수를 사용하려면 7.2 항목을 참조하세요. 이 항목에서는 SQL 변수의 정의와 사용 예제를 제공합니다. 메뉴에서 'SQL 변수' 섹션을 클릭한 후, 예제를 따라해 보시면 됩니다.

Q48: 분석 실행을 시작하려면 어떻게 해야 하나요?  
A: 분석 실행을 시작하려면 8장 '분석실행'을 클릭하세요. 이 섹션에서는 분석 실행의 기본 절차와 필요한 도구에 대한 정보를 제공합니다. 각 세부 항목을 따라가며 단계별로 진행하시면 됩니다.

Q49: 패턴 분석(컬럼 분석)은 어떻게 수행하나요?  
A: 패턴 분석을 수행하려면 8.1 항목을 클릭하세요. 이 항목에서는 패턴 분석의 정의와 함께 구체적인 실행 방법을 단계별로 설명합니다. 필요한 도구와 설정 방법을 따라 진행해 보세요.

Q50: 정규 표현식 분석(컬럼 분석)은 어떤 절차로 진행하나요?  
A: 정규 표현식 분석을 진행하려면 8.2 항목을 참조하세요. 이 항목에서는 정규 표현식의 개념과 함께 분석을 위한 구체적인 단계와 예제를 제공합니다. 메뉴에서 '정규표현식분석'을 클릭하여 내용을 확인하세요.

Q51: 개별 코드(코드 분석)를 어떻게 분석하나요?  
A: 개별 코드 분석을 하려면 8.3 항목을 클릭하세요. 이 항목에서는 개별 코드의 분석 방법과 필요한 절차를 상세히 설명합니다. 메뉴에서 '개별코드' 섹션을 찾아 실행 방법을 따라해 보세요.

Q56: 사용자정의코드 분석을 어떻게 진행하나요?  
A: 사용자정의코드 분석을 진행하려면 먼저 대시보드에 로그인한 후, 상단 메뉴에서 '분석'을 클릭합니다. 그 다음 '사용자정의코드' 항목을 선택하면 코드 분석 화면으로 이동합니다. 여기에서 분석하고자 하는 코드를 입력하고 '분석 시작' 버튼을 클릭하면 결과를 확인할 수 있습니다.

Q57: 사용빈도 분석은 어떻게 수행하나요?

Q60: 품질지표(DQI)별 조회는 어떻게 진행하나요?

Q61: 데이터 원천별 조회는 어떻게 하나요?

Q62: 사용자 정의 분석을 어떻게 수행하나요?  
A: 사용자 정의 분석을 수행하려면 대시보드에 로그인한 후, '분석' 메뉴를 클릭하고 '사용자 정의 분석'을 선택합니다. 비완성형 한글 검증 분석을 선택한 후, 분석할 데이터를 입력하고 '분석 시작' 버튼을 클릭하면 결과를 확인할 수 있습니다.

Q65: 사용자 정의 분석에서 전화번호 검증을 어떻게 수행하나요?  
A: 사용자 정의 분석을 위해 먼저 대시보드에 로그인한 후, 왼쪽 메뉴에서 "분석"을 클릭하세요. 그 다음 "사용자 정의 분석" 옵션을 선택하고 "전화번호 검증" 기능을 찾아 클릭합니다. 이후 검증할 전화번호 목록을 업로드하거나 직접 입력한 후, "검증 시작" 버튼을 눌러 분석을 진행하세요. 결과는 화면에 표시되며, 필요시 다운로드할 수 있습니다.

Q66: 추이분석을 통해 어떤 정보를 얻을 수 있나요?  
A: 추이분석을 통해 특정 데이터의 시간에 따른 변화를 시각적으로 확인할 수 있습니다. 대시보드에 로그인한 후, "분석" 메뉴에서 "추이분석"을 선택하세요. 원하는 데이터 범위를 설정하고, 분석할 지표를 선택한 후 "분석 시작" 버튼을 클릭하면 그래프 형태로 결과를 확인할 수 있습니다. 이를 통해 트렌드와 패턴을 쉽게 파악할 수 있습니다.

Q67: 계수검증 기능은 어떻게 사용하나요?  
A: 계수검증 기능을 사용하려면 대시보드에 로그인한 후, "분석" 메뉴로 이동하세요. 그 다음 "계수검증"을 선택합니다. 검증할 데이터를 입력하거나 업로드한 후, 검증 기준을 설정하고 "검증 시작" 버튼을 클릭합니다. 결과는 화면에 표시되며, 필요에 따라 추가적인 분석을 진행할 수 있습니다.

Q68: 데이터검증을 위한 절차는 무엇인가요?  
A: 데이터검증을 수행하기 위해 먼저 대시보드에 로그인한 후, 왼쪽 메뉴에서 "분석"을 클릭하세요. 그 다음 "데이터검증" 옵션을 선택합니다. 검증할 데이터 파일을 업로드하거나 직접 입력한 후, 필요한 검증 기준을 설정합니다. 이후 "검증 시작" 버튼을 클릭하면 검증 결과가 화면에 표시됩니다. 결과를 바탕으로 데이터의 정확성을 확인할 수 있습니다.

Q69: DQMiner를 사용하기 위해 필요한 시스템 요구사항은 무엇인가요?  
A DQMiner를 사용하기 위해서는 Windows 10 운영체제가 필요하며, 데이터베이스는 Oracle 12c를 사용해야 합니다. 이 두 가지 조건을 충족해야 DQMiner의 기능을 정상적으로 활용할 수 있습니다.

Q70: DQMiner를 설치하기 위해 어떤 절차를 따라야 하나요?  
A DQMiner를 설치하기 위해서는 먼저 제공된 설치 가이드를 참고해야 합니다. 설치 가이드에는 DQMiner의 다운로드 링크와 설치 과정이 상세히 설명되어 있습니다. 설치가 완료된 후에는 DQMiner를 실행하여 기본 설정을 진행해야 합니다.

Q71: 실습 환경을 구성하기 위해 DB 사용자를 어떻게 생성하나요?  
A 실습 환경을 구성하기 위해서는 두 개의 DB 사용자(개발 및 운영)를 생성해야 합니다. 먼저, https://github.com/gtoneqa/gtonesoft에 접속하여 dq_sample 하위의 SQL 파일을 다운로드합니다. 그 후, sqlplus 또는 사용 중인 DB 관리 툴을 열고, 아래의 SQL 명령어를 입력하여 계정을 생성합니다:  
```sql
CREATE USER 개발 IDENTIFIED BY 비밀번호;
CREATE USER 운영 IDENTIFIED BY 비밀번호;
GRANT ALL PRIVILEGES TO 개발;
GRANT ALL PRIVILEGES TO 운영;
```
이 명령어를 실행한 후, 다운로드한 SQL 파일 내의 스크립트를 실행하여 샘플 데이터를 생성합니다.

Q72: DQMiner에서 분석 결과를 확인하려면 어떻게 해야 하나요?  
A DQMiner에서 분석 결과를 확인하기 위해서는 웹 브라우저를 열고 DQMiner의 URL에 접속합니다. 로그인 후, 분석할 데이터베이스를 선택하고 원하는 분석 작업을 실행합니다. 분석이 완료되면 결과를 웹 화면에서 확인할 수 있습니다.

Q73: DQMiner의 기본 분석 기능은 어떤 것들이 있나요?  
A DQMiner의 기본 분석 기능에는 데이터 품질 진단, 데이터 무결성 검사, 중복 데이터 탐지 등이 포함됩니다. 사용자는 이러한 기능을 통해 데이터의 품질을 체계적으로 관리하고 개선할 수 있습니다. 각 기능은 DQMiner의 메뉴에서 선택하여 사용할 수 있습니다.

Q90: 커맨드창을 어떻게 실행하나요?  
A: Windows에서 커맨드창을 실행하려면, 먼저 '시작' 버튼을 클릭한 후 '실행' 또는 '검색' 창에 'cmd'를 입력하고 Enter 키를 누릅니다. 그러면 커맨드창이 열립니다.

Q91: sqlplus를 실행하는 명령어는 무엇인가요?  
A: 커맨드창에서 sqlplus를 실행하려면, 다음 명령어를 입력하세요: `C:\>sqlplus "/as sysdba"` 그리고 Enter 키를 누릅니다. 

Q92: 테이블스페이스를 생성하려면 어떤 명령어를 사용해야 하나요?  
A: 테이블스페이스를 생성하려면 sqlplus에 접속한 후 다음 명령어를 입력하세요: `SQL> create tablespace TS_DQ_TARGET_DEV DATAFILE 'TS_DQ_TARGET_DEV.dbf' SIZE 100M AUTOEXTEND ON;` 그리고 Enter 키를 누릅니다. 추가로 다른 테이블스페이스를 생성하려면 같은 형식으로 다음 명령어를 입력하세요: `SQL> create tablespace TS_DQ_TARGET DATAFILE 'TS_DQ_TARGET.dbf' SIZE 100M AUTOEXTEND ON;`

Q93: DB 사용자를 생성하려면 어떤 명령어를 사용해야 하나요?  
A: DB 사용자를 생성하려면 sqlplus에서 다음 명령어를 입력하세요: `SQL> create user C##DQ_TARGET_DEV identified by "DQ_TARGET_DEV" default tablespace TS_DQ_TARGET_DEV;` 그리고 Enter 키를 누릅니다. 또 다른 사용자를 생성하려면 다음 명령어를 입력하세요: `SQL> create user C##DQ_TARGET identified by "DQ_TARGET" default tablespace TS_DQ_TARGET;`

Q94: 생성한 DB 사용자에게 권한을 부여하려면 어떻게 하나요?  
A: 생성한 DB 사용자에게 권한을 부여하려면 다음 명령어를 입력하세요: `SQL> grant connect, resource, select any dictionary to C##DQ_TARGET_DEV;` 그리고 Enter 키를 누릅니다. 다른 사용자에게도 같은 권한을 부여하려면 다음 명령어를 입력하세요: `SQL> grant connect, resource, select any dictionary to C##DQ_TARGET;`

Q95: 테이블스페이스 용량이 모두 사용되도록 설정하려면 어떻게 해야 하나요?  
A: 일부 Oracle 버전에서는 기본 테이블스페이스 용량이 모두 사용되도록 허용하지 않으므로, 테이블스페이스를 'unlimited'로 설정해야 합니다. 이를 위해 다음 명령어를 입력하세요: `SQL> ALTER USER C##DQ_TARGET_DEV DEFAULT TABLESPACE TS_DQ_TARGET_DEV;` 그리고 Enter 키를 누릅니다.

Q117: TS_DQ_TARGET 테이블스페이스에 대해 무제한 쿼터를 설정하려면 어떻게 해야 하나요?

이 명령어를 실행한 후, 쿼터가 제대로 설정되었는지 확인하세요.

Q148: 데이터 품질 진단을 위한 웹 페이지에 어떻게 접속하나요?

Q150: 시스템을 추가하는 방법은 무엇인가요?

Q151: 데이터베이스를 추가하려면 어떻게 해야 하나요?  
A: 데이터베이스를 추가하려면 먼저 추가한 시스템을 선택한 후, [데이터베이스 추가] 버튼을 클릭합니다. 데이터베이스 기본정보를 입력한 후 [확인] 버튼을 클릭하여 추가를 완료합니다. 예를 들어, 데이터베이스 명은 '운영 DB'로 입력하고 설명란에는 '데이터 품질 진단 대상 데이터베이스 – 운영계'라고 기재할 수 있습니다.

Q152: 데이터베이스 연결 정보를 입력하는 방법은 무엇인가요?  
A: 데이터베이스 연결 정보를 입력하려면 추가한 데이터베이스를 선택한 후, '데이터베이스' 탭으로 이동합니다. 여기서 연결 정보를 입력하고 [연결 테스트] 버튼을 클릭하여 연결 상태를 확인할 수 있습니다. 예를 들어, 연결명은 'DQ_TARGET', 유형은 'ORACLE'로 설정할 수 있습니다.

Q158: 데이터베이스에 연결하기 위해 필요한 정보는 무엇인가요?  
A 데이터베이스에 연결하기 위해서는 다음 정보를 입력해야 합니다:  
- JDBC URL: `jdbc:oracle:thin:@127.0.0.1:1521:ORCL`  
- 접속 계정 ID: `C##DQ_TARGET`  
- 접속 계정 PW: `DQ_TARGET`  
이 정보를 입력한 후 연결 테스트를 진행하여 성공 메시지를 확인해야 합니다.

Q159: 연결 테스트 후 어떤 작업을 해야 하나요?  
A 연결 테스트가 성공적으로 완료되면, 화면에서 [저장] 버튼을 클릭하여 설정을 저장해야 합니다. 이 단계는 연결 정보를 시스템에 등록하는 중요한 과정입니다.

Q160: 진단 대상 스키마를 어떻게 설정하나요?  
A "대상 스키마 관리" 탭으로 이동한 후, 진단 대상인 “C##DQ_TARGET” 스키마를 선택합니다. 그 다음 [진단 대상 포함] 버튼을 클릭하고, 마지막으로 [저장] 버튼을 클릭하여 설정을 완료합니다.

Q161: 진단 대상 관리 목록에서 수집 작업을 어떻게 수행하나요?  
A 진단 대상 관리 목록에서 “운영 DB” 하위에 등록한 “C##DQ_TARGET” 스키마를 선택합니다. 그런 다음 [수집] 버튼을 클릭하여 진단 데이터를 수집합니다. 이 과정은 데이터 품질 진단을 위한 필수 단계입니다.

Q162: 업무를 생성하려면 어떤 절차를 따라야 하나요?  
A '진단 기준 관리' 메뉴에서 ‘업무’ 항목을 선택합니다. 그 후 [선택된 업무 하위에 업무 추가] 버튼을 클릭하고, 업무명을 입력한 후 [확인] 버튼을 클릭하여 업무를 생성합니다. 예를 들어, 업무명으로 "운영-데이터품질"을 추천합니다.

Q163: 생성한 업무에 진단 대상 테이블을 추가하는 방법은 무엇인가요?  
A 추가한 업무를 선택한 후 “진단 대상 테이블” 탭으로 이동합니다. 여기서 [진단 대상 테이블을 추가합니다.] 버튼을 클릭하면 테이블 선택 창이 나타납니다. “C##DQ_TARGET” 스키마를 선택하고 [검색] 버튼을 클릭하여 검색된 모든 테이블을 선택한 후 [확인] 버튼을 클릭합니다.

Q164: 진단 대상 테이블 추가 후 어떤 작업을 해야 하나요?  
A “진단 대상 테이블”에 추가한 테이블이 목록에 나타나는 것을 확인한 후, [저장] 버튼을 클릭하여 변경 사항을 저장합니다. 이 단계는 진단 대상 테이블을 시스템에 등록하는 중요한 과정입니다.

Q170: DQMiner에서 새로운 역할을 추가하려면 어떻게 해야 하나요?  
A 새로운 역할을 추가하기 위해서는 다음 단계를 따라주세요. 1. DQMiner의 메인 화면에서 ‘사용자/권한’ 메뉴를 클릭합니다. 2. 그 다음, ‘역할’ 메뉴를 선택합니다. 3. 화면 오른쪽 상단에 있는 [추가] 버튼을 클릭합니다. 4. 역할 추가 창이 열리면, 역할명(예: Developer)과 설명(예: IT Developer)을 입력합니다. 5. 입력이 완료되면 [확인] 버튼을 클릭하여 역할을 추가합니다.

Q171: 추가한 역할에 메뉴 권한을 부여하려면 어떻게 해야 하나요?  
A 추가한 역할에 메뉴 권한을 부여하기 위해서는 다음 단계를 따라주세요. 1. ‘사용자/권한’ 메뉴에서 ‘역할’로 이동하여 방금 추가한 역할을 클릭합니다. 2. 역할 세부 정보 화면에서 ‘메뉴 권한’ 탭을 선택합니다. 3. 해당 역할에서 사용할 메뉴(예: 진단 실행, 진단 결과, 작업 로그 등)를 체크합니다. 4. 모든 메뉴를 선택한 후, 화면 하단의 [저장] 버튼을 클릭하여 변경 사항을 저장합니다.

Q172: DQMiner에서 기본적으로 생성되는 역할은 어떤 것들이 있나요?  
A DQMiner 설치 시 기본적으로 생성되는 역할은 총 5개입니다. 이들 역할은 IT 담당자, 업무 담당자, 품질 관리자, 품질 담당자, 협업 관리자입니다. 각 역할은 특정 권한이 설정되어 있어, 사용자가 해당 역할에 맞는 기능을 수행할 수 있도록 지원합니다.

Q173: 역할을 생성할 때 어떤 정보를 입력해야 하나요?  
A 역할을 생성할 때 입력해야 하는 정보는 역할명과 설명입니다. 역할명은 해당 역할을 나타내는 이름으로, 예를 들어 'Developer'와 같이 입력할 수 있습니다. 설명은 해당 역할의 목적이나 기능을 간단히 설명하는 내용으로, 예를 들어 'IT Developer'와 같이 작성할 수 있습니다. 이 두 가지 정보를 입력한 후 [확인] 버튼을 클릭하여 역할을 추가합니다.

Q174: 메뉴 권한을 부여할 때 어떤 메뉴를 선택해야 하나요?  
A 메뉴 권한을 부여할 때 선택할 수 있는 메뉴는 사용자의 역할에 따라 다릅니다. 예를 들어, 'Developer' 역할을 위해서는 진단 실행, 진단 결과, 작업 로그, 스케줄, 보고서, 협업 등의 메뉴를 선택할 수 있습니다. 필요한 메뉴를 체크한 후 [저장] 버튼을 클릭하여 권한을 부여합니다.

Q179: 추가한 역할에 업무 권한을 어떻게 부여하나요?  
A 업무 권한을 부여하기 위해서는 다음 단계를 따라주세요. 첫째, 웹 화면에서 '업무 권한' 탭을 클릭합니다. 둘째, 추가한 역할에서 사용할 업무에 대해 '조회', '관리', '실행'의 체크박스를 선택합니다. 마지막으로, 모든 설정이 완료되면 화면 하단의 [저장] 버튼을 클릭하여 변경 사항을 저장합니다. 예를 들어, 역할 명을 'Developer'로 설정하고, 업무를 '운영-데이터품질'로 선택한 후 권한 종류로 '조회', '관리', '실행'을 체크하면 됩니다.

Q180: 추가한 역할에 보고서 권한을 어떻게 부여하나요?  
A 보고서 권한을 부여하기 위해서는 먼저 웹 화면에서 '보고서 권한' 탭을 클릭합니다. 이후, 해당 역할에 필요한 보고서 권한을 선택하고 설정합니다. 모든 권한 설정이 완료되면 화면 하단의 [저장] 버튼을 클릭하여 변경 사항을 저장합니다. 이 과정을 통해 추가한 역할에 필요한 보고서 권한을 부여할 수 있습니다.

Q181: 보고서를 체크하고 저장하는 방법은 무엇인가요?  
A 보고서를 체크하고 저장하려면, 먼저 해당 역할에서 사용할 보고서를 선택해야 합니다. 역할 명을 'Developer'로 설정한 후, 업무 별 품질 현황과 같은 보고서명을 체크합니다. 그런 다음 화면 하단에 위치한 [저장] 버튼을 클릭하여 변경 사항을 저장합니다.

Q182: 그룹을 설정하는 방법은 어떻게 되나요?  
A 그룹을 설정하려면, 먼저 '사용자/권한' 메뉴로 이동한 후 '그룹'을 선택합니다. 그 다음, 화면 상단에 있는 [추가] 버튼을 클릭하여 그룹 추가 창이 열리면, 그룹 명을 입력하고 역할을 체크합니다. 마지막으로 [확인] 버튼을 클릭하여 그룹을 추가합니다.

Q183: 사용자를 추가하는 절차는 무엇인가요?  
A 사용자를 추가하려면, '사용자/권한' 메뉴로 가서 '사용자'를 선택합니다. 그 후, 화면 상단에 있는 [추가] 버튼을 클릭하여 사용자 추가 창이 열리면, 필요한 정보를 입력합니다. 모든 정보를 입력한 후 [확인] 버튼을 클릭하여 사용자를 추가합니다.

Q184: DQMiner에 로그인하려면 어떻게 해야 하나요?  
A: DQMiner에 로그인하려면 다음 단계를 따라주세요.  
1. 우측 상단의 'DQMiner'를 클릭한 후, '로그아웃'을 선택하여 현재 세션을 종료합니다.  
2. 웹 브라우저에서 DQMiner URL(예: http://127.0.0.1:9095)로 접속합니다.  
3. 등록한 사용자 ID(예: dqdev)와 비밀번호(예: dqdev)를 입력합니다.  
4. 입력이 완료되면 [Login] 버튼을 클릭하여 로그인합니다.  

Q185: 컬럼 분석을 시작하려면 어떻게 해야 하나요?  
A: 컬럼 분석을 시작하려면 다음 단계를 따르세요.  
1. DQMiner에 로그인한 후, 상단 메뉴에서 '진단 실행'을 클릭합니다.  
2. 드롭다운 메뉴에서 '컬럼 분석'을 선택합니다.  
3. 상단 검색조건에서 '업무'와 '데이터베이스'를 선택하고, '테이블 명' 입력란에 원하는 테이블명을 입력합니다. 이때, '%' 문자를 사용하여 LIKE 검색을 할 수 있습니다.  
4. 모든 조건을 입력한 후, [컬럼 검색] 버튼을 클릭하여 검색 결과를 확인합니다.  

Q186: 분석 항목을 설정하려면 어떻게 해야 하나요?  
A: 분석 항목을 설정하려면 다음 단계를 따르세요.  
1. 검색 결과에서 분석할 항목(예: "POSITION_CODE")의 우측에 있는 [분석 항목 설정] 버튼을 클릭합니다.  
2. 분석유형에서 '통계 분석'과 '빈도 분석'을 체크한 후, [확인] 버튼을 클릭하여 설정을 완료합니다.  

Q187: 유효 값을 추가하는 방법은 무엇인가요?  
A: 유효 값을 추가하려면 다음 단계를 따라주세요.  
1. 분석 항목 설정 후, [분석 옵션 설정] 버튼을 클릭합니다.  
2. 나타나는 창에서 '유효 값' 탭을 선택합니다.  
3. 유효 값 입력란에 추가하고자 하는 값을 입력합니다(예: A, S).  
4. 모든 값을 입력한 후, [확인] 버튼을 클릭하여 설정을 완료합니다.  

Q193: POSITION_CODE 분석을 어떻게 실행하나요?  
A POSITION_CODE를 분석하기 위해서는 먼저 해당 항목을 체크한 후, 화면에서 [분석 실행] 버튼을 클릭합니다. 이렇게 하면 분석이 시작됩니다.

Q194: 진단 결과를 어떻게 확인하나요?  
A 진단 결과를 확인하려면 ‘진단 결과’ 메뉴로 이동한 후, 그 아래의 ‘컬럼 분석’ 옵션을 선택합니다. 이 메뉴에서 분석 결과를 확인할 수 있습니다.

Q195: 운영-데이터품질 메뉴에 접근하려면 어떻게 해야 하나요?  
A 운영-데이터품질 메뉴에 접근하기 위해서는 ‘업무’ 메뉴를 클릭한 후, 드롭다운 목록에서 ‘운영-데이터품질’을 선택합니다. 그 다음 [조회] 버튼을 클릭하여 관련 데이터를 검색합니다.

Q196: POSITION_CODE의 진단 상세 결과를 어떻게 열 수 있나요?  
A POSITION_CODE의 진단 상세 결과를 열려면, 검색된 결과 목록에서 “POSITION_CODE” 라인의 우측에 있는 [진단 상세 결과] 버튼을 클릭합니다. 이후 나타나는 팝업에서 [현재 창에서 열기] 버튼을 클릭하면 상세 결과를 확인할 수 있습니다.

Q197: 통계 분석 탭에서 어떤 정보를 확인할 수 있나요?  
A 통계 분석 탭에서는 직급코드에 대한 통계 데이터를 확인할 수 있습니다. 이 탭을 클릭하면 관련된 다양한 통계 정보가 표시됩니다.

Q198: 빈도분석 탭에서 오류로 표시된 값들을 어떻게 확인할 수 있나요?  
A: 빈도분석 탭에 들어가면 직급코드별 데이터의 수와 함께 유효 값으로 입력되지 않은 값들이 오류로 표시됩니다. 이를 확인하기 위해서는 먼저 해당 탭으로 이동한 후, 각 직급코드별 데이터 수치를 살펴보면 오류가 있는 값들이 강조되어 나타납니다. 오류가 있는 데이터는 일반적으로 빨간색 또는 다른 색상으로 표시되어 쉽게 식별할 수 있습니다.

Q199: 날짜분석을 수행하려면 어떤 절차를 따라야 하나요?  
A: 날짜분석을 수행하기 위해서는 다음 단계를 따라야 합니다. 첫째, 상단 메뉴에서 ‘진단 실행’을 클릭한 후, 드롭다운 메뉴에서 '컬럼 분석'을 선택합니다. 둘째, 검색조건 입력란에 원하는 항목을 입력합니다. 예를 들어, 업무 운영-데이터 품질, 데이터베이스 전체 데이터베이스, 테이블 검색 키워드로 %user를 입력합니다. 셋째, 검색결과에서 “EXPIRED_DATE” 라인의 우측에 있는 [분석항목 설정] 버튼을 클릭합니다. 여기서 분석유형에서 “통계 분석”과 “날짜 분석”을 체크한 후 [확인] 버튼을 클릭합니다. 마지막으로, [분석 옵션 설정] 버튼을 클릭하고 “유효 날짜” 탭에서 날짜유형과 유효 날짜 범위를 입력한 후 [확인] 버튼을 클릭합니다.

Q200: 분석 옵션 설정에서 유효 날짜 범위는 어떻게 입력하나요?  
A: 분석 옵션 설정에서 유효 날짜 범위를 입력하기 위해서는 먼저 [분석 옵션 설정] 버튼을 클릭한 후, 나타나는 창에서 “유효 날짜” 탭으로 이동합니다. 여기에서 날짜유형을 YYYYMMDDhh24miss 형식으로 설정하고, 유효 날짜 범위란에 시작 날짜와 종료 날짜를 입력합니다. 예를 들어, 20221108000000부터 20231231235959까지의 범위를 입력한 후 [확인] 버튼을 클릭하면 됩니다.

Q201: "EXPIRED_DATE" 분석을 어떻게 실행하나요?  
A: 먼저, 분석을 실행하기 위해 "EXPIRED_DATE" 체크박스를 선택합니다. 그 후, 화면에서 [분석 실행] 버튼을 클릭하여 분석을 시작합니다.

Q202: 분석 결과를 어떻게 확인하나요?  
A: 분석이 완료되면, 화면의 메뉴에서 ‘진단 결과’를 클릭한 후, ‘컬럼 분석’ 메뉴로 이동합니다. 여기서 분석 결과를 확인할 수 있습니다.

Q203: "EXPIRED_DATE"의 진단 상세 결과를 어떻게 조회하나요?  
A: ‘업무’ 메뉴에서 ‘운영-데이터품질’을 선택한 후, [조회] 버튼을 클릭합니다. 검색된 결과에서 "EXPIRED_DATE" 라인의 우측에 있는 [진단 상세 결과] 버튼을 클릭한 다음, [새 창에서 열기] 버튼을 클릭하여 상세 결과를 확인합니다.

Q204: "만료일"의 통계 데이터는 어디에서 확인하나요?  
A: 진단 상세 결과 창에서 "통계 분석" 탭을 클릭하면 "만료일"에 대한 통계 데이터를 확인할 수 있습니다.

Q205: 날짜 분석 탭에서 오류가 발생한 데이터를 어떻게 확인할 수 있나요?  
A: 날짜 분석 탭으로 이동한 후, 유효 날짜 범위에 해당하지 않는 데이터가 오류로 표시되는 것을 확인할 수 있습니다. 이 탭에서 오류가 발생한 데이터는 일반적으로 빨간색으로 강조 표시되며, 이를 통해 어떤 데이터가 유효하지 않은지를 쉽게 식별할 수 있습니다.

Q206: 코드 분석 기능을 사용하려면 어떻게 해야 하나요?  
A: 코드 분석 기능을 사용하기 위해서는 먼저 ‘진단 실행’ 메뉴로 이동한 후, ‘코드 분석’을 선택합니다. 그 다음, 화면에서 [추가] 버튼을 클릭하여 코드 분석에 필요한 정보를 입력하고 [확인] 버튼을 클릭합니다. 예를 들어, 코드 분석 명에는 "코드 테이블의 코드그룹 ID 분석"을 입력하고, 대상 업무는 "운영-데이터품질"을 선택합니다.

Q207: 코드 분석을 추가한 후 어떻게 편집하나요?  
A: 코드 분석을 추가한 후, 추가한 항목을 더블 클릭하면 편집 화면으로 이동합니다. 여기서 진단 대상을 설정한 후, 변경사항을 저장하고 [분석 실행] 버튼을 클릭하여 분석을 실행할 수 있습니다. 진단 데이터베이스와 진단 기준을 적절히 설정하는 것을 잊지 마세요.

Q208: 진단 데이터베이스와 진단 기준은 어떻게 설정하나요?  
A: 진단 데이터베이스는 "운영-DQ_TARGET" 하위의 "운영 DB"를 선택하고, 진단 기준에는 TB_CD_GRP.CD_GRP_ID를 입력합니다. 진단 대상을 설정할 때는 TB_CD.CD_GRP_ID를 선택하여 분석할 데이터를 명확히 지정합니다. 이렇게 설정한 후에는 [저장] 버튼을 클릭하여 변경사항을 저장합니다.

Q210: 코드 분석 결과에서 미사용 데이터와 오류 데이터를 어떻게 확인하나요?  
A: 먼저, ‘진단 결과’ 메뉴에서 ‘코드 분석’을 선택합니다. 그 다음 ‘업무’ 메뉴에서 ‘운영-데이터품질’을 클릭하고, [조회] 버튼을 눌러 분석 항목을 조회합니다. 이후 분석 항목의 우측에 있는 [진단 상세 결과] 버튼을 클릭한 후, 나타나는 팝업에서 [현재 창에서 열기] 버튼을 클릭합니다. 코드 분석 상세 결과 화면에서 미사용 데이터는 TB_CD_GRP 테이블에는 존재하지만 TB_CD 테이블에는 존재하지 않는 CD_GRP_ID로 표시되며, 오류 데이터는 TB_CD_GRP 테이블에는 존재하지 않고 TB_CD 테이블에만 존재하는 CD_GRP_ID로 나타납니다.

Q211: 관계 분석을 수행하려면 어떻게 해야 하나요?  
A: 관계 분석을 수행하기 위해서는 먼저 ‘진단 실행’ 메뉴로 이동하여 ‘관계 분석’을 선택합니다. 이후 화면에서 [추가] 버튼을 클릭하여 필요한 정보를 입력한 후, 입력이 완료되면 [확인] 버튼을 클릭하여 분석을 시작합니다. 이 과정을 통해 두 개의 관련 테이블 간의 데이터 무결성을 검증할 수 있습니다.

Q212: 관계 분석을 추가한 후 어떻게 편집할 수 있나요?  
A: 추가한 관계 분석을 편집하려면 먼저 해당 관계 분석 항목을 더블 클릭하여 편집화면으로 이동합니다. 여기서 진단 대상을 설정한 후, 오른쪽 상단에 있는 [저장] 버튼을 클릭합니다. 이후 [분석 실행] 버튼을 클릭하여 분석을 시작할 수 있습니다.

Q213: 진단 데이터베이스와 진단 테이블을 설정하는 방법은 무엇인가요?  
A: 진단 데이터베이스를 설정하려면 편집화면에서 "운영-DQ_TARGET" 하위의 "운영 DB"를 선택합니다. 진단 테이블은 진단기준 TB_CD_GRP를 선택하고, 진단대상으로 TB_CD를 선택합니다. 이를 통해 필요한 진단 데이터베이스와 테이블을 설정할 수 있습니다.

Q214: 진단 컬럼을 설정하는 방법은 무엇인가요?  
A: 진단 컬럼을 설정하려면 편집화면에서 진단 기준으로 CD_GRP_ID를 선택하고, 진단대상으로도 CD_GRP_ID를 선택합니다. 이 두 컬럼을 설정한 후, [저장] 버튼을 클릭하여 변경 사항을 저장합니다.

Q215: 진단 결과를 조회하려면 어떻게 해야 하나요?  
A: 진단 결과를 조회하려면 먼저 ‘진단 결과’ 메뉴에서 ‘관계 분석’을 선택합니다. 그 다음, ‘업무’ 메뉴에서 ‘운영-데이터품질’을 선택하고, 오른쪽에 있는 [조회] 버튼을 클릭합니다. 이렇게 하면 해당 진단 결과를 확인할 수 있습니다.

Q216: 참조 무결성의 상세 결과를 어떻게 확인하나요?  
A: 참조 무결성의 항목을 클릭하면 아래에 상세 결과가 조회됩니다. 이 상세 결과를 더 자세히 확인하려면 우측에 있는 [진단 상세 결과] 버튼을 클릭한 후, [새 창으로 열기] 버튼을 클릭하여 새로운 창에서 결과를 확인할 수 있습니다.

Q218: 관계 분석 상세 결과 화면에서 참조 무결성에 위배되는 데이터를 어떻게 확인할 수 있나요?  
A: 관계 분석 상세 결과 화면에 접근하려면 먼저 시스템에 로그인한 후, 상단 메뉴에서 '분석'을 클릭합니다. 그 다음 '관계 분석'을 선택하고, 원하는 분석 항목을 클릭하여 상세 결과 화면으로 이동합니다. 이 화면에서 참조 무결성에 위배되는 데이터는 경고 메시지 또는 오류 목록으로 표시됩니다. 이를 통해 어떤 데이터가 무결성 규칙을 위반하고 있는지 확인할 수 있습니다.

Q219: 업무 규칙을 설정하고 분석하는 방법은 무엇인가요?  
A: 업무 규칙을 설정하려면 먼저 '진단 실행' 메뉴로 이동한 후 '업무 규칙'을 선택합니다. 화면에서 [추가] 버튼을 클릭하여 새로운 업무 규칙 정보를 입력합니다. 규칙 이름, 상태, 품질 지표 등을 입력한 후 [저장] 버튼을 클릭하여 규칙을 저장합니다. 진단 대상의 컬럼은 진단 SQL을 입력한 후 [대상 컬럼 동기화] 버튼을 클릭하면 자동으로 입력됩니다. Alias가 설정된 경우 빈칸은 수동으로 입력해야 합니다.

Q220: 진단 SQL을 작성하는 방법은 무엇인가요?  
A: 진단 SQL을 작성하려면 업무 규칙 설정 화면에서 '진단 SQL' 입력란에 SQL 쿼리를 입력합니다. 예를 들어, 사용자 정보를 확인하기 위해 SELECT 문을 작성할 수 있습니다. SQL 문을 입력한 후에는 [저장] 버튼을 클릭하여 규칙을 저장하고, 이후 분석을 실행하여 결과를 확인합니다. SQL 문 작성 시 필요한 테이블과 컬럼을 정확히 지정하는 것이 중요합니다.

Q221: 대상 컬럼 동기화 버튼은 어떤 기능을 하나요?  
A: 대상 컬럼 동기화 버튼은 진단 SQL을 입력한 후 자동으로 진단 대상 컬럼을 업데이트하는 기능을 제공합니다. 이 버튼을 클릭하면 SQL 쿼리에서 참조된 컬럼들이 자동으로 입력란에 반영됩니다. 만약 Alias가 설정된 경우, 빈칸으로 표시되는 부분은 사용자가 직접 입력해야 하므로 주의가 필요합니다.

Q222: 업무 규칙 예시를 어떻게 설정하나요?  
A: 업무 규칙 예시를 설정하려면 '업무 규칙' 화면에서 [추가] 버튼을 클릭한 후, 다음과 같은 정보를 입력합니다: 업무 규칙 명(예: "사용자 정보에는 사내메일(gtone)이 등록되어 있어야 한다"), 상태(예: "사용중"), 품질 지표(DQI)(예: "Format 유효성"), 담당자(예: "㈜ 개발자"), 진단 대상 테이블 및 컬럼 정보를 입력합니다. 모든 정보를 입력한 후 [저장] 버튼을 클릭하여 규칙을 저장합니다.

Q233: SQL 쿼리에서 특정 이메일 도메인을 제외하려면 어떻게 해야 하나요?  
A: 이메일 도메인을 제외하려면 `WHERE` 절에 `EMAIL NOT LIKE '%@gtone.co.kr'` 조건을 추가해야 합니다. 예를 들어, 전체 사용자 수를 세기 위한 SQL 쿼리는 다음과 같이 작성할 수 있습니다:  
```sql
SELECT COUNT(USER_ID)  
FROM C##DQ_TARGET.TB_USER  
WHERE EMAIL NOT LIKE '%@gtone.co.kr';
```
이 쿼리는 `C##DQ_TARGET.TB_USER` 테이블에서 `@gtone.co.kr` 도메인을 가진 이메일을 가진 사용자 수를 제외한 총 사용자 수를 반환합니다.

Q234: SQL 쿼리에서 특정 조건을 만족하는 레코드의 수를 세는 방법은 무엇인가요?  
A: 특정 조건을 만족하는 레코드의 수를 세기 위해 `SELECT COUNT(column_name)` 구문을 사용합니다. 예를 들어, `USER_ID`를 기준으로 사용자 수를 세고 싶다면 다음과 같이 쿼리를 작성할 수 있습니다:  
```sql
SELECT COUNT(USER_ID)  
FROM C##DQ_TARGET.TB_USER  
WHERE [조건];
```
여기서 `[조건]` 부분에 원하는 필터링 조건을 추가하면 됩니다.

Q235: SQL 쿼리에서 조인을 사용하는 방법은 무엇인가요?  
A: SQL 쿼리에서 조인을 사용하려면 `JOIN` 키워드를 사용하여 두 개 이상의 테이블을 연결합니다. 예를 들어, `POSITION_CODE`와 `CLASS` 테이블을 조인하려면 다음과 같은 형식을 사용할 수 있습니다:  
```sql
SELECT *  
FROM C##DQ_TARGET.TB_USER AS USER  
JOIN CLASS ON USER.POSITION_CODE = CLASS.CD_ID;
```
이 쿼리는 `TB_USER` 테이블과 `CLASS` 테이블을 `POSITION_CODE`와 `CD_ID`를 기준으로 조인하여 결과를 반환합니다.

Q236: SQL 쿼리에서 특정 컬럼만 선택하고 싶을 때는 어떻게 하나요?  
A: 특정 컬럼만 선택하려면 `SELECT` 문 뒤에 원하는 컬럼명을 나열하면 됩니다. 예를 들어, `USER_ID`와 `EMAIL` 컬럼만 선택하고 싶다면 다음과 같이 쿼리를 작성합니다:  
```sql
SELECT USER_ID, EMAIL  
FROM C##DQ_TARGET.TB_USER;
```
이 쿼리는 `TB_USER` 테이블에서 `USER_ID`와 `EMAIL` 컬럼의 데이터만 반환합니다.

Q248: 업무 규칙 편집 화면에서 분석 실행 버튼은 어떻게 찾나요?  
A: 업무 규칙 편집 화면에 들어가면 상단 또는 중앙에 위치한 버튼들 중에서 '분석 실행' 버튼을 찾아 클릭하세요. 이 버튼은 일반적으로 다른 기능 버튼들과 함께 배치되어 있습니다.

Q249: 진단 결과 메뉴로 어떻게 이동하나요?  
A: 분석 실행 버튼을 클릭한 후, 화면의 왼쪽 사이드바 또는 상단 메뉴에서 '진단 결과'를 찾아 클릭합니다. 그 다음, 드롭다운 메뉴에서 '업무 규칙'을 선택하여 해당 메뉴로 이동합니다.

Q250: 업무 규칙을 조회하려면 어떤 절차를 따라야 하나요?  
A: '업무 규칙' 메뉴를 선택한 후, '업무' 카테고리에서 '운영-데이터품질'을 클릭합니다. 그 다음, 화면에 나타나는 [조회] 버튼을 클릭하여 업무 규칙 목록을 불러옵니다.

Q251: 진단 상세 결과를 어떻게 확인하나요?  
A: 조회된 업무 규칙 목록에서 원하는 업무 규칙의 우측에 위치한 [진단 상세 결과] 버튼을 클릭합니다. 그러면 새로운 창이 열리며, 해당 업무 규칙의 상세 진단 결과를 확인할 수 있습니다.

Q252: 사내메일(gtone)을 사용하지 않는 사용자 정보를 어떻게 확인하나요?  
A: 업무 규칙 상세 결과 화면이 열리면, 해당 화면에서 사내메일(gtone)을 사용하지 않는 사용자 목록이 표시됩니다. 이 정보를 스크롤하여 확인하거나, 필요한 경우 검색 기능을 활용하여 특정 사용자를 찾을 수 있습니다.

Q253: 스케줄 설정을 어떻게 시작하나요?  
A 스케줄 설정을 시작하려면 먼저 웹 화면에서 ‘스케줄’ 메뉴를 선택합니다. 그 다음, 화면 오른쪽 상단에 위치한 [추가] 버튼을 클릭하여 스케줄 정보를 입력합니다. 입력이 완료되면 [확인] 버튼을 클릭하여 다음 단계로 진행합니다.

Q254: 스케줄 설정 시 어떤 정보를 입력해야 하나요?  
A 스케줄 설정 시 입력해야 할 정보는 스케줄 명과 대상 업무입니다. 예를 들어, 스케줄 명으로는 "DQ_TARGET 분석"을, 대상 업무로는 "운영-데이터품질"을 입력할 수 있습니다. 이러한 정보를 입력한 후 [확인] 버튼을 클릭하여 저장합니다.

Q255: 스케줄 편집 화면에서 어떤 항목을 설정해야 하나요?  
A 스케줄 편집 화면에서는 차수와 사용 예를 설정해야 합니다. 설정이 완료되면 화면 하단에 있는 [저장] 버튼을 클릭하여 변경 사항을 저장합니다.

Q256: 스케줄 상세 설정은 어떻게 하나요?  
A 스케줄 상세 설정을 하려면 스케줄 설정 화면에서 [상세 설정] 버튼을 클릭합니다. 이후 스케줄 상세 설정 화면에서 [추가] 버튼을 클릭하여 4개의 작업 그룹을 등록합니다.

Q257: 작업 그룹을 등록한 후에는 어떻게 해야 하나요?  
A 작업 그룹을 등록한 후에는 각 작업 그룹을 더블 클릭하여 앞서 등록한 분석 항목들을 추가해야 합니다. 예를 들어, 작업 그룹 명으로 "컬럼 분석 그룹"을 설정하고, 유형으로 "컬럼 분석"을 선택한 후, 분석할 컬럼을 지정합니다.

Q259: 코드 분석 그룹을 생성하려면 어떤 정보를 입력해야 하나요?  
A: 코드 분석 그룹을 생성하기 위해서는 다음 정보를 입력해야 합니다. 첫째, '작업 그룹 명'에는 '코드 분석 그룹'을 입력합니다. 둘째, '유형'에는 '코드 분석'을 선택합니다. 마지막으로, '코드 분석 코드 테이블의 코드그룹 ID'를 입력해야 합니다. 이 정보를 모두 입력한 후 '저장' 버튼을 클릭하여 그룹을 생성합니다.

Q260: 관계 분석 그룹을 만들고 싶습니다. 어떤 절차를 따라야 하나요?  
A: 관계 분석 그룹을 만들기 위해서는 다음 단계를 따르세요. 첫째, '작업 그룹 명'에 '관계 분석 그룹'을 입력합니다. 둘째, '유형'에는 '관계 분석'을 선택합니다. 셋째, '관계 분석 코드 그룹 테이블과 코드 테이블의 참조 무결성'에 필요한 정보를 입력합니다. 모든 정보를 입력한 후 '저장' 버튼을 클릭하여 그룹을 생성합니다.

Q261: 업무 규칙 그룹을 설정할 때 어떤 내용을 입력해야 하나요?  
A: 업무 규칙 그룹을 설정하기 위해서는 다음과 같은 정보를 입력해야 합니다. 첫째, '작업 그룹 명'에 '업무 규칙 그룹'을 입력합니다. 둘째, '유형'에는 '업무 규칙'을 선택합니다. 셋째, '업무규칙' 항목에는 '사용자 정보에는 사내메일(gtone)이 등록되어 있어야 한다'와 같은 규칙을 입력합니다. 모든 정보를 입력한 후 '저장' 버튼을 클릭하여 그룹을 설정합니다.

Q262: 레이아웃을 어떻게 적용하나요?  
A: 상단 메뉴에서 [레이아웃 적용] 버튼을 클릭합니다. 이후 작업 그룹에 마우스를 가져가면 손 모양의 커서가 나타납니다. 이 상태에서 클릭한 후, 원하는 다른 그룹으로 드래그하여 화살표가 생기면 그 위치에 드랍합니다. 

Q263: 설정한 내용을 어떻게 저장하나요?  
A: 레이아웃 적용 후, 상단의 [저장] 버튼을 클릭하여 설정한 내용을 저장합니다. 저장이 완료되면 상단 메뉴에서 [스케줄]을 클릭하여 스케줄 목록으로 돌아갑니다.

Q264: 등록한 스케줄을 분석하려면 어떻게 해야 하나요?  
A: 스케줄 목록에서 분석하고자 하는 등록된 스케줄을 선택한 후, 상단의 [분석] 버튼을 클릭합니다. 이렇게 하면 선택한 스케줄에 대한 분석이 실행됩니다.

Q265: 분석 결과는 어디서 확인할 수 있나요?  
A: 분석을 실행한 후, 상단 메뉴에서 [작업 로그] 버튼을 클릭하면 작업 로그 화면으로 이동합니다. 여기서 실행한 분석의 진행 상황 및 결과를 확인할 수 있습니다.

Q266: DQMiner에 로그인하려면 어떻게 해야 하나요?  
A DQMiner에 로그인하려면 다음 단계를 따르세요. 먼저 웹 브라우저를 실행한 후, 주소창에 DQMiner URL(예: http://127.0.0.1:9095)을 입력하고 Enter 키를 누릅니다. 로그인 화면이 나타나면 관리자용 ID와 패스워드를 입력합니다. 기본 값은 ID가 'DQMiner'이고 패스워드도 'DQMiner'입니다. 모든 정보를 입력한 후, [Login] 버튼을 클릭하여 로그인합니다.

Q267: 진단대상을 등록하려면 어떤 절차를 따라야 하나요?  
A 진단대상을 등록하기 위해서는 다음 절차를 따르세요. 먼저 DQMiner에 로그인한 후, 좌측 메뉴에서 ‘진단 대상 관리’를 클릭합니다. 그 다음, [시스템을 추가] 버튼을 클릭하여 시스템 정보를 입력합니다. 예를 들어, 시스템 명에 '개발-DQ_TARGET_DEV'를 입력하고 설명란에 '데이터 품질 진단 대상 시스템 – 개발계'를 입력한 후 [확인] 버튼을 클릭합니다. 추가한 시스템을 선택한 후, [데이터베이스 추가] 버튼을 클릭하여 데이터베이스 기본정보를 입력하고 [확인] 버튼을 클릭합니다. 마지막으로 추가한 데이터베이스를 선택하고 “데이터베이스” 탭에서 연결정보를 입력한 후 [연결 테스트] 버튼을 클릭합니다.

Q268: 데이터베이스 추가 시 어떤 정보를 입력해야 하나요?  
A 데이터베이스를 추가할 때는 다음 정보를 입력해야 합니다. 데이터베이스 명에는 '개발 DB'를 입력하고, 설명란에는 '데이터 품질 진단 대상 데이터베이스 – 개발계'를 입력합니다. 이 정보를 입력한 후 [확인] 버튼을 클릭하여 데이터베이스를 등록합니다.

Q269: 연결 테스트는 어떻게 수행하나요?  
A 연결 테스트를 수행하려면 먼저 추가한 데이터베이스를 선택한 후, ‘데이터베이스’ 탭으로 이동합니다. 그곳에서 연결정보를 입력한 후, [연결 테스트] 버튼을 클릭합니다. 이 버튼을 클릭하면 DQMiner가 입력한 연결정보를 바탕으로 데이터베이스와의 연결 상태를 확인합니다. 연결이 성공적으로 이루어지면 확인 메시지가 나타납니다.

Q278: Oracle 데이터베이스에 연결하기 위한 정보는 무엇인가요?  
A: Oracle 데이터베이스에 연결하기 위해 필요한 정보는 다음과 같습니다. 연결명은 "DQ_TARGET_DEV"이며, 유형은 "ORACLE"입니다. 연결 정보는 "jdbc:oracle:thin:@127.0.0.1:1521:ORCL"입니다. 접속 계정 ID는 "C##DQ_TARGET_DEV"이고, 비밀번호는 "DQ_TARGET_DEV"입니다. 이 정보를 사용하여 데이터베이스에 연결할 수 있습니다.

Q279: 데이터베이스 연결 후 저장 버튼을 클릭하려면 어떻게 해야 하나요?  
A: 데이터베이스 연결이 성공적으로 이루어지면 화면에 [저장] 버튼이 활성화됩니다. 이 버튼을 클릭하여 연결 정보를 저장합니다. 연결이 성공했는지 확인하는 방법은 오류 메시지가 나타나지 않는지 체크하는 것입니다.

Q280: "대상 스키마 관리" 탭으로 이동하려면 어떻게 해야 하나요?  
A: 데이터베이스 연결 후, 상단 메뉴에서 "대상 스키마 관리" 탭을 클릭합니다. 이후 진단 대상인 "C##DQ_TARGET_DEV" 스키마를 선택한 후, 화면에서 [진단 대상 포함] 버튼을 클릭합니다. 마지막으로 [저장] 버튼을 클릭하여 변경 사항을 저장합니다.

Q281: 진단 대상 관리 목록에서 "C##DQ_TARGET_DEV" 스키마를 선택하고 수집하려면 어떻게 하나요?  
A: 진단 대상 관리 목록으로 이동한 후, "개발 DB" 하위에서 진단 대상으로 등록한 "C##DQ_TARGET_DEV" 스키마를 찾아 선택합니다. 그런 다음 화면에서 [수집] 버튼을 클릭하여 데이터를 수집합니다.

Q282: 업무를 생성하고 진단대상 테이블을 매핑하려면 어떤 절차를 따라야 하나요?  
A: 업무를 생성하려면 먼저 '진단 기준 관리' 메뉴에서 '업무'를 선택합니다. 그 후 [선택된 업무 하위에 업무 추가] 버튼을 클릭하고, 업무명을 입력한 후 [확인] 버튼을 클릭합니다. 예를 들어, 업무명으로 "개발-데이터품질"을 입력할 수 있습니다.

Q283: 진단 대상 테이블을 추가하려면 어떻게 해야 하나요?  
A: 추가한 업무를 선택한 후, "진단 대상 테이블" 탭으로 이동합니다. 여기서 [진단 대상 테이블을 추가합니다.] 버튼을 클릭하면 테이블 선택 창이 나타납니다. "테이블 선택" 창에서 "C##DQ_TARGET_DEV" 스키마를 선택하고 [검색] 버튼을 클릭하여 검색된 모든 테이블을 선택한 후 [확인] 버튼을 클릭합니다.

Q291: 진단 대상 테이블에 테이블을 추가한 후 어떤 절차를 따라야 하나요?  
A: 진단 대상 테이블에 테이블을 추가한 후, 화면 오른쪽 상단에 위치한 [저장] 버튼을 클릭하여 변경 사항을 저장합니다.

Q292: 업무 권한 설정을 하려면 어떤 메뉴를 선택해야 하나요?  
A: 화면 왼쪽 메뉴에서 ‘사용자/권한’ 항목을 클릭한 후, 하위 메뉴에서 ‘역할’을 선택합니다.

Q293: "Developer" 역할에 대한 업무 권한을 어떻게 설정하나요?  
A: ‘사용자/권한’ > ‘역할’ 메뉴에서 “Developer” 역할을 클릭한 후, ‘업무 권한’ 탭으로 이동합니다. 여기서 “개발-데이터품질” 업무의 조회, 관리, 실행 체크박스를 선택하고, 화면 하단의 [저장] 버튼을 클릭하여 설정을 완료합니다.

Q294: CTQ란 무엇인가요?  
A: CTQ는 데이터 품질을 측정하는 컬럼의 논리적 그룹을 의미하며, 데이터 품질 관리에 중요한 기준 정보를 제공합니다.

Q295: CTQ 그룹을 추가하려면 어떤 절차를 따라야 하나요?  
A: ‘진단 기준 관리’ 메뉴에서 ‘핵심 관리 항목(CTQ)’을 선택한 후, 화면 상단의 [그룹 추가] 버튼을 클릭합니다. CTQ 그룹 정보를 입력한 후 [확인] 버튼을 클릭하여 그룹을 추가합니다.

Q296: 추가한 CTQ 그룹에 컬럼을 어떻게 추가하나요?  
A: 추가한 CTQ 그룹을 선택한 후, 화면 상단의 [CTQ 추가] 버튼을 클릭합니다. 이후 CTQ 기본정보를 입력하고 [확인] 버튼을 클릭하여 컬럼을 추가합니다.

Q306: 부가 정보는 어떻게 작성해야 하나요?  
A: 부가 정보는 사용자가 충분히 이해할 수 있도록 명확하고 구체적으로 작성해야 합니다. 예를 들어, 각 항목의 의미와 사용 목적을 설명하고, 필요한 경우 예시를 포함하여 사용자가 쉽게 이해할 수 있도록 합니다.

Q307: "대상 컬럼" 탭에 접근하려면 어떻게 해야 하나요?  
A: 웹 화면에서 "대상 컬럼" 탭을 클릭하여 해당 메뉴로 이동합니다. 이 탭에서 추가할 컬럼을 선택할 수 있습니다.

Q308: 컬럼을 추가하려면 어떤 절차를 따라야 하나요?  
A: "대상 컬럼" 탭에서 [추가] 버튼을 클릭한 후, "컬럼 선택" 상단의 검색 조건에서 업무와 스키마를 선택하고 테이블 명을 입력합니다. 입력 후 [검색] 버튼을 클릭하여 원하는 컬럼을 찾습니다.

Q309: LIKE 검색을 사용하려면 어떻게 해야 하나요?  
A: 테이블명 입력 시 '%' 문자를 사용하여 LIKE 검색을 수행할 수 있습니다. 예를 들어, 테이블명에 '%FESTIVAL'을 입력하면 'FESTIVAL'이 포함된 모든 테이블을 검색할 수 있습니다.

Q310: 검색 결과에서 컬럼을 선택하는 방법은 무엇인가요?  
A: 검색 결과에서 원하는 컬럼의 물리명(예: "HOMEPAGE")을 체크한 후, [확인] 버튼을 클릭하여 선택을 완료합니다.

Q311: 선택한 컬럼을 저장하려면 어떻게 해야 하나요?  
A: "대상 컬럼" 탭에서 [저장] 버튼을 클릭하여 선택한 컬럼을 저장합니다. 이 단계에서 저장이 완료되면 추가한 컬럼이 시스템에 반영됩니다.

Q312: DQI란 무엇인가요?  
A: DQI는 데이터의 품질 수준을 측정하기 위한 지표로, 데이터의 완전성, 정확성, 일관성 등을 평가하는 데 사용됩니다.

Q313: DQI를 정의하려면 어떤 절차를 따라야 하나요?  
A: ‘진단 기준 관리’ 메뉴에서 ‘품질 지표(DQI)’를 선택한 후, 상위 지표로 "완전성"을 선택하고 [추가] 버튼을 클릭합니다. 이후 품질 지표명을 입력하고 [확인] 버튼을 클릭하여 정의를 완료합니다.

Q314: 품질 지표명을 어떻게 입력하나요?  
A: 품질 지표명을 입력할 때는 명확하고 구체적인 이름을 사용해야 합니다. 예를 들어, "적시 완전성"과 같이 데이터 품질을 잘 나타내는 이름을 입력합니다.

Q315: 사용자 정의 코드는 무엇을 위한 것인가요?  
A: 사용자 정의 코드는 데이터베이스에 정의되지 않은 코드를 분석하기 위해 사용됩니다. 이를 통해 데이터의 품질을 높이고, 분석을 용이하게 할 수 있습니다.

Q320: 사용자 정의 코드를 등록하려면 어떻게 해야 하나요?  
A 사용자 정의 코드를 등록하기 위해서는 다음 단계를 따라주세요.  
1. 웹 화면에서 ‘진단 기준 관리’ 메뉴를 클릭한 후, ‘사용자 정의 코드’ 옵션을 선택합니다.  
2. 화면 오른쪽 상단에 있는 [추가] 버튼을 클릭합니다.  
3. 코드명과 설명을 입력합니다. 예를 들어, 코드명에 '휴가코드', 설명에 '휴가코드를 정의합니다.'를 입력한 후 [확인] 버튼을 클릭합니다.  
4. 이제 ‘기본정보 > 코드값’ 메뉴로 이동하여 [추가] 버튼을 클릭합니다.  
5. 코드값과 코드값 명을 입력합니다. 예를 들어, 코드값에 'Sick leave', 코드값명에 '병가'를 입력한 후 [저장] 버튼을 클릭합니다.  
6. 동일한 방법으로 추가 코드값과 코드값 명을 등록합니다. 예시로는 코드값 'Maternity leave'와 코드값명 '출산휴가', 코드값 'Half day leave'와 코드값명 '반차', 코드값 'Annual leave'와 코드값명 '연차'를 입력할 수 있습니다.  

Q321: 사용자 정의 코드 등록 시 코드값은 어떻게 추가하나요?  
A 코드값을 추가하려면 다음 단계를 따라주세요.  
1. 사용자 정의 코드 등록 후, ‘기본정보 > 코드값’ 메뉴로 이동합니다.  
2. 화면에서 [추가] 버튼을 클릭합니다.  
3. 코드값 입력란에 원하는 값을 입력합니다. 예를 들어, 'Sick leave'를 입력하고, 코드값명 입력란에 '병가'를 입력합니다.  
4. 모든 정보를 입력한 후, [저장] 버튼을 클릭하여 추가한 코드값을 저장합니다.  
5. 추가로 필요한 코드값이 있다면, 같은 방법으로 계속해서 입력하고 저장하면 됩니다.  

Q322: 사용자 정의 코드의 예시로 어떤 것들이 있나요?  
A 사용자 정의 코드의 예시로는 다음과 같은 항목들이 있습니다.  
1. 코드명: '휴가코드', 설명: '휴가코드를 정의합니다.'  
2. 코드값: 'Sick leave', 코드값명: '병가'  
3. 코드값: 'Maternity leave', 코드값명: '출산휴가'  
4. 코드값: 'Half day leave', 코드값명: '반차'  
5. 코드값: 'Annual leave', 코드값명: '연차'  
이 외에도 필요에 따라 다양한 코드명과 코드값을 정의할 수 있습니다.  

Q323: 사용자 정의 코드 등록 후 수정은 어떻게 하나요?  
A 사용자 정의 코드를 수정하려면 다음 단계를 따라주세요.  
1. ‘진단 기준 관리’ 메뉴에서 ‘사용자 정의 코드’ 옵션을 선택합니다.  
2. 수정하고자 하는 코드명을 찾아 클릭하여 선택합니다.  
3. 선택한 코드의 세부 정보가 나타나면, 수정하고 싶은 항목(코드명 또는 설명)을 변경합니다.  
4. 변경이 완료되면 [확인] 버튼을 클릭하여 수정 내용을 저장합니다.  
5. 코드값을 수정하고 싶다면, ‘기본정보 > 코드값’ 메뉴로 이동하여 수정할 코드값을 선택한 후, 필요한 내용을 수정하고 [저장] 버튼을 클릭합니다.  

Q324: 사용자 정의 코드와 코드값의 차이는 무엇인가요?  
A 사용자 정의 코드는 특정한 카테고리를 정의하는 이름으로, 예를 들어 '휴가코드'와 같은 항목입니다. 반면, 코드값은 그 카테고리 내에서 구체적인 항목을 나타내며, 예를 들어 'Sick leave'나 'Annual leave'와 같은 구체적인 휴가 유형을 의미합니다. 즉, 사용자 정의 코드는 큰 틀을 제공하고, 코드값은 그 틀 안에서 세부적인 내용을 정의합니다.

Q325: 사용자 정의 분석을 설정하려면 어떻게 해야 하나요?  
A 사용자 정의 분석을 설정하기 위해서는 다음 단계를 따라주세요.  
1. 시스템 메뉴에서 ‘진단 기준 관리’를 선택합니다.  
2. 그 다음 ‘사용자 정의 분석’ 메뉴를 클릭합니다.  
3. Default Group을 선택한 후, [편집] 버튼을 클릭합니다.  
4. 시스템 목록에서 “운영-DQ_TARGET” 시스템을 선택하고 [확인] 버튼을 클릭합니다.  

Q326: 데이터 원천을 등록하려면 어떤 절차를 따라야 하나요?  
A 데이터 원천을 등록하기 위해서는 아래의 절차를 따르세요.  
1. ‘진단 부가 항목 관리’ 메뉴로 이동하여 ‘데이터 원천’을 선택합니다.  
2. [데이터 원천 그룹 추가] 버튼을 클릭합니다.  
3. 데이터 원천 그룹 코드와 데이터 원천 그룹 명을 입력한 후 [확인] 버튼을 클릭합니다. 예를 들어, 데이터 원천 그룹 코드로 "CITY_COUNTY"와 데이터 원천 그룹 명으로 "지역"을 입력할 수 있습니다.  
4. ‘기본정보 > 데이터 원천 코드’로 이동하여 [추가] 버튼을 클릭합니다.  
5. 데이터 원천 코드와 데이터 원천 명을 입력하고 [저장] 버튼을 클릭합니다. 예를 들어, 데이터 원천 코드로 "강원도"와 데이터 원천 명으로 "강원도"를 입력할 수 있습니다.  

Q327: 데이터 원천 그룹을 추가할 때 어떤 정보를 입력해야 하나요?  
A 데이터 원천 그룹을 추가할 때는 다음 정보를 입력해야 합니다.  
1. 데이터 원천 그룹 코드: 예를 들어, "CITY_COUNTY"와 같은 고유한 코드입니다.  
2. 데이터 원천 그룹 명: 예를 들어, "지역"과 같이 그룹의 내용을 설명하는 이름입니다.  
이 정보를 입력한 후 [확인] 버튼을 클릭하여 그룹을 추가할 수 있습니다.  

Q328: 진단 실행 시 사용할 SQL 변수는 어떻게 관리하나요?  
A 진단 실행 시 사용할 SQL 변수는 ‘진단 부가 항목’ 메뉴에서 관리할 수 있습니다.  
1. ‘진단 부가 항목 관리’ 메뉴로 이동합니다.  
2. 필요한 SQL 변수를 추가하거나 수정하여 진단 실행 시 사용할 수 있도록 설정합니다.  
이 과정에서 구체적인 SQL 변수의 내용은 사용자의 요구에 따라 다를 수 있습니다.  

Q329: 오류 데이터 원천을 찾기 위해 데이터 원천을 어떻게 활용하나요?  
A 오류 데이터 원천을 찾기 위해 데이터 원천은 다음과 같이 활용됩니다.  
1. 등록된 데이터 원천을 기반으로 컬럼 분석이나 업무 규칙을 수행합니다.  
2. 분석 결과를 통해 분류된 데이터를 확인하고, 오류가 발생한 데이터 원천을 식별합니다.  
예를 들어, “공공 데이터 포털”의 “지역별 축제 정보”를 사용하여 데이터를 취합하고 지역별로 분리하여 오류를 확인할 수 있습니다.

Q335: 데이터 원천 코드를 등록하려면 어떻게 해야 하나요?  
A 데이터 원천 코드를 등록하려면 다음 단계를 따르세요. 1. 해당 시스템의 데이터 원천 등록 메뉴로 이동합니다. 2. '데이터 원천 코드' 입력란에 원하는 코드를 입력합니다. 예를 들어, '전라남도'를 입력할 수 있습니다. 3. '데이터 원천 명' 입력란에도 동일하게 '전라남도'를 입력합니다. 4. 모든 정보를 입력한 후, 저장 버튼을 클릭하여 등록을 완료합니다. 동일한 방법으로 다른 데이터 원천도 추가할 수 있습니다.

Q336: SQL 변수를 등록하려면 어떤 절차를 따라야 하나요?  
A SQL 변수를 등록하려면 다음 절차를 따르세요. 1. 시스템 메뉴에서 ‘진단 부가 항목 관리’로 이동한 후, ‘SQL 변수’ 메뉴를 선택합니다. 2. 화면에서 [추가] 버튼을 클릭합니다. 3. 변수명을 입력합니다. 예를 들어, '#v_year'를 입력할 수 있습니다. 4. 데이터 타입을 선택합니다. 이때 'CHAR' 또는 'NUMBER' 중 하나를 선택할 수 있습니다. 5. 모든 정보를 입력한 후 [확인] 버튼을 클릭하여 등록을 완료합니다.

Q337: 분석 실행을 위해 로그인하는 방법은 무엇인가요?  
A 분석 실행을 위해 로그인하려면 다음 단계를 따르세요. 1. 우측 상단에 있는 DQMiner 아이콘을 클릭합니다. 2. 드롭다운 메뉴에서 [로그아웃]을 선택하여 현재 세션을 종료합니다. 3. 웹 브라우저를 열고 DQMiner URL(예: http://127.0.0.1:9095)로 접속합니다. 4. 로그인 화면에서 권한이 설정된 아이디와 비밀번호를 입력하여 로그인합니다.

Q343: 패턴 분석을 시작하려면 어떻게 해야 하나요?  
A 패턴 분석을 시작하려면 먼저 로그인해야 합니다. 로그인하기 위해서는 등록한 ID와 패스워드(dqdev/dqdev)를 입력한 후, [Login] 버튼을 클릭합니다. 로그인 후, 상단 메뉴에서 ‘진단 실행’을 선택하고, 그 다음 ‘컬럼 분석’ 메뉴를 클릭합니다.

Q344: 컬럼 분석을 위한 테이블 검색은 어떻게 하나요?  
A 컬럼 분석을 위한 테이블 검색을 하려면, 상단 검색조건에서 업무와 데이터베이스를 선택한 후, 테이블 명을 입력합니다. 이때, 테이블명 입력 시 ‘%’ 문자를 사용하여 LIKE 검색을 할 수 있습니다. 예를 들어, 업무를 ‘운영-데이터품질’로 설정하고 데이터베이스를 ‘전체데이터베이스’로 선택한 후, 테이블 검색 키워드에 ‘%user’를 입력하고 [컬럼 검색] 버튼을 클릭합니다.

Q345: 분석할 항목을 설정하려면 어떻게 해야 하나요?  
A 분석할 항목을 설정하려면, 검색 결과에서 “USER_NM” 라인의 우측에 있는 [분석항목 설정] 버튼을 클릭합니다. 이후 분석유형에서 “패턴 분석”을 체크한 후 [확인] 버튼을 클릭하여 설정을 완료합니다.

Q346: 유효 패턴을 추가하는 방법은 무엇인가요?  
A 유효 패턴을 추가하려면, [분석 옵션 설정] 버튼을 클릭한 후 “유효 패턴” 탭으로 이동합니다. 여기에서 원하는 유효 패턴 값을 추가하고 [확인] 버튼을 클릭하여 설정을 완료합니다. 예를 들어, 유효 패턴으로 ‘HHH’를 추가할 수 있습니다.

Q347: 패턴 분석을 실행하려면 어떻게 해야 하나요?  
A 패턴 분석을 실행하려면, 먼저 “USER_NM” 항목을 체크한 후 [분석 실행] 버튼을 클릭합니다. 이렇게 하면 설정한 패턴 분석이 실행됩니다. 분석이 완료되면 ‘진단 결과’ 메뉴로 이동하여 결과를 확인할 수 있습니다.

Q348: 데이터 품질을 확인하기 위해 어떻게 사용자 이름의 패턴 분석 결과를 조회할 수 있나요?  
A: 먼저, 상단 메뉴에서 ‘업무’를 클릭한 후 ‘운영-데이터품질’을 선택하세요. 그 다음, [조회] 버튼을 클릭하여 검색 결과를 확인합니다. 검색된 목록에서 “USER_NM” 라인의 우측에 있는 [진단 상세 결과] 버튼을 클릭한 후, 팝업 창에서 [현재 창에서 열기] 버튼을 클릭하면 사용자 이름의 패턴 분석 결과를 확인할 수 있습니다.

Q349: 패턴 분석 결과에서 유효 패턴이 무엇인지 확인하려면 어떻게 해야 하나요?  
A: 패턴 분석 결과를 확인한 후, “패턴 분석” 탭으로 이동하세요. 여기에서 사용자 이름의 패턴 분석 결과를 확인할 수 있으며, 유효 패턴으로 등록된 HHH에 대해서는 오류 여부가 체크되지 않는 것을 확인할 수 있습니다.

Q350: 정규표현식 분석을 수행하려면 어떤 절차를 따라야 하나요?  
A: 정규표현식 분석을 수행하기 위해서는 먼저 ‘진단 실행’ 메뉴에서 ‘컬럼 분석’을 선택하세요. 상단의 검색 조건에서 업무와 데이터베이스를 선택한 후, 테이블 명을 입력하고 [컬럼 검색] 버튼을 클릭합니다. 테이블명 입력 시 ‘%’ 문자를 사용하여 LIKE 검색을 할 수 있습니다.

Q351: 특정 컬럼에 대한 정규표현식 분석을 설정하려면 어떻게 해야 하나요?  
A: 검색 결과에서 “USER_IP” 라인의 우측에 있는 [분석항목 설정] 버튼을 클릭하세요. 분석유형에서 “정규표현식 분석”을 체크한 후 [확인] 버튼을 클릭합니다. 그 다음, [분석 옵션 설정] 버튼을 클릭하고 “유효 정규표현식” 탭에 필요한 값을 추가한 후 [확인] 버튼을 클릭하면 설정이 완료됩니다.

Q352: 유효 정규표현식의 예시는 무엇인가요?  
A: 유효 정규표현식의 예시로는 다음과 같은 패턴을 사용할 수 있습니다: ^((25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)\.){3}(25[0-5]|2[0-4][0-9]|[01]?[0-9][0-9]?)$ 이 정규표현식은 IP 주소 형식을 검증하는 데 사용됩니다.

Q353: USER_IP 분석을 실행하려면 어떻게 해야 하나요?  
A USER_IP를 체크한 후, 화면에서 [분석 실행] 버튼을 클릭하여 분석을 시작합니다.

Q354: 컬럼 분석 결과를 어떻게 확인하나요?  
A ‘진단 결과’ 메뉴로 이동한 후, ‘컬럼 분석’ 항목을 선택합니다. 그 다음 ‘업무’에서 ‘운영-데이터품질’을 클릭하고, [컬럼 분석 진단 결과 검색] 버튼을 눌러 검색합니다. 검색된 USER_IP 라인의 우측에 있는 [진단 상세 결과] 버튼을 클릭한 후, [현재 창에서 열기] 버튼을 클릭합니다.

Q355: 정규표현식 분석 결과는 어디에서 확인하나요?  
A 분석 결과 페이지에서 “정규표현식 분석” 탭을 클릭하면 생성자아이피의 정규표현식 분석 결과를 확인할 수 있습니다. 이곳에서 유효한 Ipv4 형태가 아닌 값이 오류 데이터로 조회됨을 확인할 수 있습니다.

Q356: 개별 코드를 분석하려면 어떻게 해야 하나요?  
A 개별 코드를 분석하기 위해서는 다음 단계를 따라주세요. 먼저, 웹 화면에서 ‘진단 실행’ 메뉴를 클릭한 후, ‘코드 분석’ 옵션을 선택합니다. 그 다음, [추가] 버튼을 클릭하여 필요한 정보를 입력하고 [확인] 버튼을 클릭합니다. 여기서 코드 분석 명은 사용자 테이블의 부서코드를 확인하고, 대상 업무는 운영-데이터품질, 분석 유형은 개별코드로 설정합니다. 추가한 코드 분석을 더블 클릭하여 편집화면으로 이동한 후, 진단 대상과 진단 추가 조건을 선택하고 [저장] 버튼을 클릭합니다. 마지막으로 [분석 실행] 버튼을 클릭하여 분석을 시작합니다.

Q357: 코드 분석을 위한 진단 데이터베이스는 어떻게 설정하나요?  
A 코드 분석을 위한 진단 데이터베이스를 설정하려면, 편집화면에서 진단 데이터베이스를 “운영-DQ_TARGET” 하위의 “운영 DB”로 선택해야 합니다. 또한, 진단 기준은 TB_DEP_INFO.DEP_ID로 설정하고, 진단 대상은 TB_USER.DEP_ID로 지정합니다. 이 모든 설정을 완료한 후, [저장] 버튼을 클릭하여 변경 사항을 저장합니다.

Q358: 코드 분석 결과를 확인하려면 어떤 절차를 따라야 하나요?  
A 코드 분석 결과를 확인하기 위해서는 ‘진단 결과’ 메뉴에서 ‘코드 분석’ 옵션을 선택합니다. 이후 ‘업무’ 항목에서 운영-데이터품질을 선택하고 [조회] 버튼을 클릭합니다. 목록에서 분석하고자 하는 ‘개별코드’를 클릭한 후, 분석 항목의 우측에 있는 [진단 상세 결과] 버튼을 클릭합니다. 마지막으로 [현재 창에서 열기] 버튼을 클릭하여 결과를 확인할 수 있습니다.

Q360: 코드 분석 상세 결과 화면에서 오류 데이터를 어떻게 확인하나요?  
A: 코드 분석 상세 결과 화면에서 오류 데이터를 확인하려면, 먼저 코드 분석을 실행한 후 결과 화면으로 이동합니다. 여기서 TB_DEP_INFO 테이블을 확인하여 DEP_ID가 7인 부서가 존재하지 않는 경우, 해당 데이터가 오류로 표시됩니다. 오류 데이터를 확인하려면 결과 목록에서 관련 항목을 클릭하여 세부 정보를 확인하세요.

Q361: 사용자정의 코드를 분석하는 방법은 무엇인가요?  
A: 사용자정의 코드를 분석하기 위해서는 다음 단계를 따르세요. 첫째, 상단 메뉴에서 ‘진단 실행’을 클릭한 후 ‘코드 분석’을 선택합니다. 둘째, 화면에서 [추가] 버튼을 클릭하여 분석 정보를 입력합니다. 예를 들어, 코드 분석 명은 "사용자 테이블의 휴가코드 분석", 대상 업무는 "운영-데이터품질", 분석 유형은 "사용자 정의 코드"로 설정합니다. 마지막으로 [확인] 버튼을 클릭하여 정보를 저장합니다.

Q362: 추가한 코드 분석을 편집하려면 어떻게 해야 하나요?  
A: 추가한 코드 분석을 편집하려면, 코드 분석 목록에서 해당 항목을 더블 클릭합니다. 그러면 편집 화면으로 이동하게 됩니다. 여기서 진단 대상과 진단 옵션을 선택한 후, 변경 사항을 [저장]하고 [분석 실행] 버튼을 클릭하여 분석을 시작하세요.

Q363: 분석 실행 시 어떤 데이터베이스를 선택해야 하나요?  
A: 분석 실행 시 진단 데이터베이스로 "운영-DQ_TARGET" 하위의 "운영 DB"를 선택해야 합니다. 이 데이터베이스를 선택하면 분석 대상인 TB_USER 테이블의 OFF_DUTY_CODE에 대한 분석을 수행할 수 있습니다.

Q364: NULL 오류를 제외하는 방법은 무엇인가요?  
A: NULL 오류를 제외하려면, 분석 실행 화면에서 진단 대상 설정 시 "NULL 오류 제외" 옵션을 선택해야 합니다. 이 옵션을 활성화하면 분석 과정에서 NULL 값을 가진 데이터는 제외되고, 나머지 데이터에 대해서만 분석이 진행됩니다.

Q366: 코드 분석 결과에서 오류 데이터를 어떻게 확인하나요?  
A: 코드 분석 결과를 확인하려면 다음 단계를 따라주세요. 먼저, 메뉴에서 ‘진단 결과’를 선택한 후 ‘코드 분석’ 메뉴로 이동합니다. 그 다음, ‘업무’ 항목에서 ‘운영-데이터품질’을 선택하고 [조회] 버튼을 클릭합니다. 이후 ‘사용자 정의 코드’를 클릭한 후, 분석항목의 우측에 있는 [진단 상세 결과] 버튼을 클릭합니다. 마지막으로, [현재 창에서 열기] 버튼을 클릭하면 코드 분석 상세 결과 화면이 열리며, 여기에서 오류 데이터를 확인할 수 있습니다. 오류 데이터는 미사용 코드와 오류 코드로 구분되어 표시됩니다. 미사용 코드는 등록되었으나 사용되지 않은 코드이며, 오류 코드는 등록되지 않고 사용된 코드입니다.

Q367: 사용빈도 분석을 시작하려면 어떻게 해야 하나요?  
A 사용빈도 분석을 시작하려면, 먼저 ‘진단 실행’ 메뉴로 이동한 후 ‘관계 분석’ 메뉴를 선택하세요. 그 다음, 화면에서 [추가] 버튼을 클릭하여 필요한 정보를 입력하고 [확인] 버튼을 클릭합니다. 예를 들어, 분석할 관계 분석 명을 'TB_CD 테이블의 TB_USER.POSITION_CODE에 대한 사용빈도 분석'으로 설정하고, 대상 업무를 '운영-데이터품질'로 지정한 후 분석 유형을 '사용빈도'로 선택합니다.

Q368: 추가한 관계 분석을 어떻게 편집하나요?  
A 추가한 관계 분석을 편집하려면, 목록에서 해당 관계 분석 항목을 더블 클릭하세요. 그러면 편집화면으로 이동하게 됩니다. 여기서 진단 대상을 선택하고, 진단 추가 조건을 입력한 후 [저장] 버튼을 클릭합니다. 이후 [분석 실행] 버튼을 클릭하여 분석을 진행할 수 있습니다. 진단 데이터베이스는 “운영-DQ_TARGET” 하위의 “운영 DB”로 설정하고, 진단 테이블 및 컬럼을 적절히 지정해야 합니다.

Q369: 진단 결과를 어떻게 확인하나요?  
A 진단 결과를 확인하려면, ‘진단 결과’ 메뉴로 이동한 후 ‘관계 분석’ 메뉴를 선택하세요. 그 다음, ‘업무’ 항목에서 ‘운영-데이터품질’을 선택하고 [조회] 버튼을 클릭합니다. 그러면 사용빈도 분석 결과가 나타납니다.

Q370: 사용빈도 분석 결과를 어떻게 확인할 수 있나요?  
A 사용빈도 분석 결과를 확인하려면, ‘사용빈도’ 그리드에서 원하는 결과를 클릭하세요. 하단 목록에서 ‘TB_CD 테이블의 TB_USER.POSITION_CODE에 대한 사용빈도 분석’ 항목을 찾아 우측에 있는 [진단 상세 결과] 버튼을 클릭합니다. 이후 [현재 창에서 열기] 버튼을 클릭하면 상세 결과를 확인할 수 있습니다.

Q372: 업무 규칙을 설정하려면 어떻게 해야 하나요?  
A 업무 규칙을 설정하기 위해서는 다음 단계를 따르세요. 1. 웹 화면에서 ‘진단 실행’ 메뉴를 클릭한 후, 드롭다운 목록에서 ‘업무 규칙’을 선택합니다. 2. 화면 우측에 있는 [추가] 버튼을 클릭합니다. 3. 분석을 위한 업무 규칙 정보를 입력합니다. 이때, 진단 대상 컬럼은 진단 SQL을 입력한 후 [대상 컬럼 동기화] 버튼을 클릭하면 자동으로 입력됩니다. 4. 모든 정보를 입력한 후 [저장] 버튼을 클릭하여 업무 규칙을 저장합니다.

Q373: 진단 대상 컬럼은 어떻게 설정하나요?  
A 진단 대상 컬럼을 설정하려면, 먼저 진단 SQL을 입력해야 합니다. 진단 실행 화면에서 SQL 입력란에 원하는 SQL 쿼리를 작성한 후, [대상 컬럼 동기화] 버튼을 클릭하세요. 이 버튼을 클릭하면, SQL 쿼리에서 지정한 컬럼들이 자동으로 대상 컬럼으로 설정됩니다.

Q374: 업무 규칙 명에 포함해야 할 정보는 무엇인가요?  
A 업무 규칙 명에는 반드시 "문화축제 정보"와 "홈페이지 정보"가 포함되어야 합니다. 이는 업무 규칙을 명확히 이해하고 추적하기 위해 필수적인 요소입니다.

Q375: 품질 지표(DQI)에는 어떤 항목을 설정해야 하나요?  
A 품질 지표(DQI)에는 "적시 완전성"을 설정해야 합니다. 이는 데이터의 정확성과 완전성을 평가하는 중요한 지표입니다.

Q376: 업무 규칙의 상태는 어떻게 설정하나요?  
A 업무 규칙의 상태는 "사용중"으로 설정해야 합니다. 이를 통해 해당 업무 규칙이 현재 활성화되어 있음을 나타냅니다.

Q377: 진단 대상 테이블은 어떻게 설정하나요?  
A 진단 대상 테이블은 "TB_CULTURAL_FESTIVAL"로 설정해야 합니다. 이 테이블은 문화축제와 관련된 데이터를 포함하고 있으며, 진단 기준을 체크하는 데 사용됩니다.

Q378: 진단 SQL의 예시는 어떻게 되나요?  
A 진단 SQL의 예시는 다음과 같습니다:  
```sql
SELECT CITY_COUNTY, ID, NAME, CONTENTS, HOMEPAGE FROM TB_CULTURAL_FESTIVAL;
```
이 SQL 쿼리는 TB_CULTURAL_FESTIVAL 테이블에서 CITY_COUNTY, ID, NAME, CONTENTS, HOMEPAGE 컬럼의 데이터를 선택합니다.

Q389: 문화 축제 데이터에서 특정 지역의 정보를 어떻게 조회할 수 있나요?  
A: 문화 축제 데이터에서 특정 지역의 정보를 조회하려면, 먼저 SQL 쿼리를 작성해야 합니다. 예를 들어, 다음과 같은 쿼리를 사용할 수 있습니다:  
```sql
SELECT CITY_COUNTY, COUNT(ID)  
FROM C##DQ_TARGET.TB_CULTURAL_FESTIVAL  
WHERE HOMEPAGE IS NULL  
GROUP BY CITY_COUNTY;
```  
이 쿼리를 실행하면, HOMEPAGE가 NULL인 문화 축제의 지역별 카운트를 확인할 수 있습니다.

Q390: 업무 규칙 편집에서 데이터 원천 그룹을 설정하려면 어떻게 해야 하나요?  
A: 업무 규칙 편집에서 데이터 원천 그룹을 설정하려면, 먼저 ‘업무 규칙 편집’ 메뉴로 이동합니다. 그런 다음, ‘부가 정보’ 탭을 클릭하고, 옵션에서 데이터 원천 그룹을 “지역”으로 선택합니다. 마지막으로 [저장] 버튼을 클릭하여 변경 사항을 저장합니다.

Q391: CTQ 별 진단 결과를 어떻게 조회하나요?  
A: CTQ 별 진단 결과를 조회하려면, ‘진단결과’ 메뉴에서 ‘핵심 관리 항목(CTQ)’을 선택합니다. 그 후, ‘핵심 관리 항목(CTQ)’ 하위 메뉴에서 ‘최종 사용자 영향도.부가정보기재’를 선택하고 [조회] 버튼을 클릭하여 결과를 확인할 수 있습니다.

Q392: 분석 실행 버튼은 어디에서 찾을 수 있나요?  
A: 분석 실행 버튼은 ‘업무 규칙 편집’ 화면에서 찾을 수 있습니다. 해당 화면을 열고, 화면의 하단 또는 상단에 위치한 [분석 실행] 버튼을 클릭하면 분석을 시작할 수 있습니다.

Q399: TB_CULTURAL_FESTIVAL의 홈페이지 정보는 어떻게 확인하나요?  
A: TB_CULTURAL_FESTIVAL의 홈페이지 정보를 확인하려면, 먼저 해당 페이지로 이동한 후 우측에 위치한 [컬럼 상세] 버튼을 클릭하세요. 그러면 연관 진단으로 “문화축제 정보에 홈페이지 정보는 반드시 포함되어야 한다.”라는 메시지가 나타납니다. 이 메시지를 통해 홈페이지 정보가 포함되어 있는지 확인할 수 있습니다.

Q400: 홈페이지 정보가 포함되어 있지 않은 경우 어떻게 확인하나요?  
A: 홈페이지 정보가 포함되어 있지 않은 경우, “문화축제 정보에 홈페이지 정보는 반드시 포함되어야 한다.”라는 진단 메시지의 우측에 있는 [진단 상세 결과] 버튼을 클릭하세요. 그러면 업무 규칙 상세 결과 화면이 열리며, 여기서 홈페이지 정보가 없는 오류 데이터가 조회되는 것을 확인할 수 있습니다.

Q401: DQI 별 진단 결과는 어떻게 조회하나요?  
A: DQI 별 진단 결과를 조회하려면, 먼저 관련 메뉴로 이동한 후 DQI 관련 옵션을 선택하세요. 그 후, 원하는 DQI 항목을 선택하여 진단 결과를 확인할 수 있습니다. 구체적인 메뉴 위치는 시스템의 사용자 인터페이스에 따라 다를 수 있으므로, 해당 메뉴를 찾아보시기 바랍니다.

Q406: 품질 지표(DQI)에서 '완전성.적시완전성'을 조회하려면 어떻게 해야 하나요?  
A: 먼저, 웹 화면에서 '진단결과' 메뉴를 클릭한 후, '품질 지표(DQI)'를 선택합니다. 그 다음, '완전성.적시완전성' 항목을 클릭하고 [조회] 버튼을 눌러주세요. 그러면 “문화축제 정보에 홈페이지 정보는 반드시 포함되어야 한다.”라는 결과가 조회됩니다.

Q407: '문화축제 정보에 홈페이지 정보는 반드시 포함되어야 한다.'의 진단 상세 결과를 어떻게 확인하나요?  
A: '완전성.적시완전성' 조회 결과에서 “문화축제 정보에 홈페이지 정보는 반드시 포함되어야 한다.” 항목의 우측에 있는 [진단 상세 결과] 버튼을 클릭하세요. 그러면 업무 규칙 상세 결과 화면에서 해당 항목의 분석 결과를 확인할 수 있습니다.

Q408: 데이터 원천 별 진단 결과를 조회하려면 어떻게 해야 하나요?  
A: 웹 화면에서 '진단결과' 메뉴를 클릭한 후, '데이터원천'을 선택합니다. 그 다음, '데이터원천'에서 “지역” 항목을 클릭하고 [조회] 버튼을 눌러주세요. 그러면 '업무 규칙'에 “문화축제 정보에 홈페이지 정보는 반드시 포함되어야 한다.”라는 결과가 조회됩니다.

Q411: 문화축제 정보에 홈페이지 정보가 포함되어 있는지 어떻게 확인하나요?  
A: 문화축제 정보에 홈페이지 정보가 반드시 포함되어 있는지 확인하려면, 먼저 업무 규칙 상세 결과 화면으로 이동해야 합니다. 이를 위해 ‘진단 상세 결과’ 버튼을 클릭합니다. 이후 나타나는 화면에서 “문화축제 정보에 홈페이지 정보는 반드시 포함되어야 한다.”의 분석 결과를 확인할 수 있습니다.

Q412: 오류 데이터의 원천 값을 변경하려면 어떻게 해야 하나요?  
A: 오류 데이터의 원천 값을 변경하려면, 먼저 오류 데이터 하단에 위치한 원천 값 변경 옵션을 찾습니다. 해당 옵션에서 원하는 원천 값을 선택한 후, 변경을 완료하면 원천별로 오류 데이터를 조회할 수 있는 화면이 나타납니다.

Q413: 비완성형 한글 검증 분석을 수행하려면 어떤 절차를 따라야 하나요?  
A: 비완성형 한글 검증 분석을 수행하려면, 먼저 ‘진단 실행’ 메뉴로 이동한 후 ‘사용자 정의 분석’ 옵션을 선택합니다. 그 다음, 상단에 있는 검색 조건에서 업무와 데이터베이스를 선택하고, 테이블 명을 입력한 후 [컬럼 검색] 버튼을 클릭합니다. 이때 테이블명 입력 시 ‘%’ 문자를 사용하여 LIKE 검색이 가능합니다. 예를 들어, 업무는 ‘운영-데이터품질’, 데이터베이스는 ‘전체데이터베이스’, 테이블 검색 키워드는 ‘%user’로 입력할 수 있습니다.

Q414: 사용자 정의 분석에서 특정 분석 항목을 설정하려면 어떻게 해야 하나요?  
A: 사용자 정의 분석에서 특정 분석 항목을 설정하려면, 먼저 검색 결과에서 “USER_NM” 라인을 찾습니다. 그 라인의 우측에 있는 [분석항목 설정] 버튼을 클릭한 후, 나타나는 옵션에서 “비완성형 한글 검증”을 체크합니다. 마지막으로 [확인] 버튼을 클릭하여 설정을 완료하면 됩니다.

Q415: 사용자 이름의 비완성형 한글 검증을 어떻게 수행하나요?  
A: 사용자 이름의 비완성형 한글 검증을 수행하려면, 먼저 ‘USER_NM’ 항목을 체크한 후, 화면에서 [분석 실행] 버튼을 클릭하여 분석을 시작하세요. 분석이 완료되면 ‘진단 결과’ 메뉴로 이동하여 ‘사용자 정의 분석’을 선택합니다. 그 다음, ‘업무’ 항목에서 ‘운영-데이터품질’을 클릭하고 [조회] 버튼을 눌러 검색된 “USER_NM” 라인의 우측에 있는 [진단 상세 결과] 버튼을 클릭하면, “비완성형 한글 검증” 탭에서 결과를 확인할 수 있습니다.

Q416: 전화번호 검증은 어떻게 진행하나요?  
A: 전화번호 검증을 진행하기 위해서는 먼저 ‘진단 실행’ 메뉴에서 '사용자 정의 분석'을 선택해야 합니다. 그 후, 해당 메뉴에서 전화번호 검증을 위한 분석을 실행하면 됩니다. 분석이 완료되면 결과를 확인할 수 있습니다.

Q417: 전화번호 검증을 어떻게 실행하나요?  
A 전화번호 검증을 실행하려면 먼저 상단 검색조건에서 '업무'를 '운영-데이터품질'로 설정하고 '데이터베이스'를 '전체데이터베이스'로 선택한 후, '테이블 명' 입력란에 '%user'를 입력합니다. 그 다음 [컬럼 검색] 버튼을 클릭하여 검색 결과를 확인합니다. 검색 결과에서 “CELL_PHONE” 라인의 우측에 있는 [분석항목 설정] 버튼을 클릭하고, 사용자 정의 분석에서 “전화번호 검증”을 체크한 후 [확인] 버튼을 누릅니다. 이후 “CELL_PHONE”을 체크하고 [분석 실행] 버튼을 클릭하여 분석을 실행합니다. 마지막으로 ‘진단 결과’ > ‘사용자 정의 분석’ 메뉴로 이동하여 '업무'를 '운영-데이터품질'로 선택하고 [조회] 버튼을 클릭한 후, 전화번호 검증을 클릭하여 결과를 확인합니다.

Q418: LIKE 검색을 사용하는 방법은 무엇인가요?  
A LIKE 검색을 사용하려면 상단 검색조건에서 '업무'와 '데이터베이스'를 선택한 후, '테이블 명' 입력란에 검색하고자 하는 키워드를 입력합니다. 이때 '%' 문자를 사용하여 부분 일치를 검색할 수 있습니다. 예를 들어, '%user'를 입력하면 'user'가 포함된 모든 테이블명을 검색할 수 있습니다. 입력 후 [컬럼 검색] 버튼을 클릭하여 결과를 확인하세요.

Q419: 분석 실행 후 진단 결과를 어떻게 확인하나요?  
A 분석 실행 후 진단 결과를 확인하려면 '진단 결과' 메뉴로 이동한 다음, '사용자 정의 분석'을 선택합니다. 이후 '업무'에서 '운영-데이터품질'을 선택하고 [조회] 버튼을 클릭합니다. 그 다음, 전화번호 검증 항목을 클릭하여 관련된 진단 결과를 확인할 수 있습니다. 

Q420: 분석 항목 설정에서 전화번호 검증을 체크하는 방법은?  
A 분석 항목 설정에서 전화번호 검증을 체크하려면, 먼저 검색 결과에서 “CELL_PHONE” 라인의 우측에 있는 [분석항목 설정] 버튼을 클릭합니다. 그러면 사용자 정의 분석 목록이 나타나는데, 여기서 “전화번호 검증” 항목을 찾아 체크한 후 [확인] 버튼을 클릭하면 됩니다. 

Q421: 검색 결과에서 특정 테이블을 찾는 방법은?  
A 검색 결과에서 특정 테이블을 찾으려면, 상단 검색조건에서 '업무'와 '데이터베이스'를 설정한 후, '테이블 명' 입력란에 원하는 키워드를 입력하고 [컬럼 검색] 버튼을 클릭합니다. 검색 결과에서 원하는 테이블명을 찾아 클릭하여 추가적인 분석을 진행할 수 있습니다.

Q422: 전화번호 검증 결과는 어떻게 확인하나요?  
A: 전화번호 검증 결과를 확인하려면, 먼저 "전화번호 검증" 탭으로 이동하세요. 해당 탭에서 입력된 전화번호의 검증 결과를 확인할 수 있습니다. 검증 결과는 각 전화번호 옆에 표시되며, 유효성 여부를 쉽게 파악할 수 있습니다.

Q423: 추이 분석을 설정하려면 어떤 절차를 따라야 하나요?  
A: 추이 분석을 설정하기 위해서는 다음 단계를 따라주세요.  
1. 메뉴에서 ‘진단 실행’을 클릭한 후 ‘다중 갭 분석’을 선택하세요.  
2. 화면에서 [추가] 버튼을 클릭하여 다중갭분석 정보를 입력합니다. 예를 들어, 분석 명칭을 "사용자 등록은 0~2명 증감해야 한다"로 설정하고, 대상 업무를 "운영-데이터품질"로 선택하세요. 상태는 "사용중"으로 설정하고, 분석 유형을 "추이분석"으로 선택합니다. 오류 판단 최소값과 최대값을 각각 0과 2로 입력한 후 [저장] 버튼을 클릭합니다.

Q424: 분석 항목을 추가하는 방법은 무엇인가요?  
A: 분석 항목을 추가하려면, 먼저 분석 항목 섹션에서 [추가] 버튼을 클릭하세요. 이후 분석 항목 정보를 입력합니다. 대상 테이블을 "사용자 정보 증감율"로 설정하고, 대상 컬럼에는 "C##DQ_TARGET.TB_USER"를 입력하세요. 조회 SQL에는 "SELECT COUNT(USER_NM) FROM TB_USER WHERE SUBSTR(CRE_DT,1,4) = #v_year"를 입력하고, SQL 변수에는 변수명 "#v_year"와 할당값 "2014"를 설정합니다. 기본 사용 여부는 체크하지 않은 상태로 두고 [저장] 버튼을 클릭하세요.

Q425: 추이 분석 결과를 확인하기 위해 스케줄 등록은 어떻게 하나요?  
A: 추이 분석 결과를 확인하기 위해 스케줄을 등록하려면, 다중 갭 분석 편집 화면에서 [스케줄 등록] 버튼을 클릭하세요. 이후 스케줄 설정을 완료하면, 설정한 주기에 따라 자동으로 추이 분석 결과를 확인할 수 있습니다.

Q429: 새로운 스케줄을 등록하려면 어떻게 해야 하나요?  
A 새로운 스케줄을 등록하려면 먼저 스케줄 등록 화면으로 이동합니다. 여기서 ‘새로운 스케줄에 등록’ 옵션을 체크한 후, 스케줄 명을 입력합니다. 예를 들어, "추이분석을 위한 스케줄"이라고 입력할 수 있습니다. 그 다음, 대상 업무를 선택하고 [확인] 버튼을 클릭하여 등록을 완료합니다.

Q430: 스케줄 상세 설정은 어떻게 하나요?  
A 스케줄 상세 설정을 하려면 먼저 스케줄 설정 메뉴로 이동합니다. 그 후 ‘스케줄 상세 설정’ 옵션을 선택하고 필요한 정보를 입력합니다. 모든 설정이 완료되면 [저장] 버튼을 클릭하여 변경 사항을 저장합니다.

Q431: 등록한 스케줄을 확인하려면 어떻게 해야 하나요?  
A 등록한 스케줄을 확인하려면 스케줄 목록으로 이동합니다. 목록에서 "추이분석을 위한 스케줄"을 찾아 더블 클릭합니다. 이렇게 하면 해당 스케줄의 상세 정보를 확인할 수 있습니다.

Q432: 스케줄 설정 정보를 입력할 때 주의할 점은 무엇인가요?  
A 스케줄 설정 정보를 입력할 때는 각 항목에 대한 정확한 정보를 입력하는 것이 중요합니다. 예를 들어, 차수와 같은 필수 항목을 정확히 입력한 후, 모든 정보를 확인한 다음 [저장] 버튼을 클릭하여 설정을 완료해야 합니다.

Q433: 차수 자동 증가 기능을 사용하려면 어떻게 해야 하나요?  
A 차수 자동 증가 기능을 사용하려면 먼저 웹 화면에서 '분석' 버튼을 클릭합니다. 이후 진단 차수와 설정 값을 변경하고 저장한 뒤, 분석을 반복 실행합니다. 예를 들어, 진단 차수를 2로 설정하고 SQL 변수의 설정 값을 2012로 입력한 후 저장합니다. 그 다음, 진단 차수를 3으로 변경하고 설정 값을 2013으로 다시 입력하여 저장합니다.

Q434: 진단 결과를 확인하려면 어떤 절차를 따라야 하나요?  
A 진단 결과를 확인하려면 먼저 '진단 결과' 메뉴로 이동한 후 '다중 갭 분석'을 선택합니다. 그 다음, '업무' 메뉴에서 '운영-데이터품질'을 선택하고 [조회] 버튼을 클릭합니다. 이렇게 하면 진단 결과를 조회할 수 있습니다.

Q435: 추이 분석의 진단 결과를 어떻게 확인하나요?  
A 추이 분석의 진단 결과를 확인하려면 먼저 '진단 결과' > '다중 갭 분석' 메뉴로 이동하여 조회한 결과에서 원하는 진단 결과를 클릭합니다. 이후 하단에 나타나는 “사용자 등록은 0~2 명 증감 해야한다”라는 라인의 우측에 있는 [진단 상세 결과] 버튼을 클릭하면 상세 결과를 확인할 수 있습니다.

Q437: 계수 검증을 어떻게 설정하나요?  
A 계수 검증을 설정하기 위해서는 먼저 ‘진단 실행’ 메뉴로 이동합니다. 그 다음 ‘다중 갭 분석’ 메뉴를 선택합니다. 화면에서 [추가] 버튼을 클릭하여 분석을 위한 다중 갭 분석 정보를 입력합니다. 예를 들어, 다중 갭 분석 명에는 "운영계와 개발계의 코드테이블의 코드는 그룹별로 동일"이라고 입력하고, 대상 업무는 "운영-데이터품질"로 설정합니다. 상태는 "사용중"으로 선택하고, 분석 유형은 "계수검증"으로 설정합니다. 마지막으로, 분석 설정에서 비교 기준을 "전체비교"로, 동일 데이터 제외는 "아니오"로, 오차 범위 유형은 "수치"로 설정한 후 [저장] 버튼을 클릭하여 완료합니다.

Q438: 다중 갭 분석 정보를 입력할 때 어떤 항목을 포함해야 하나요?  
A 다중 갭 분석 정보를 입력할 때는 다음과 같은 항목을 포함해야 합니다. 첫째, 다중 갭 분석 명에는 분석의 목적이나 내용을 간단히 설명하는 문구를 입력합니다. 예를 들어, "운영계와 개발계의 코드테이블의 코드는 그룹별로 동일"이라고 작성할 수 있습니다. 둘째, 대상 업무는 "운영-데이터품질"로 설정합니다. 셋째, 상태는 "사용중"으로 선택하고, 넷째, 분석 유형은 "계수검증"으로 설정합니다. 마지막으로 분석 설정에서 비교 기준은 "전체비교", 동일 데이터 제외는 "아니오", 오차 범위 유형은 "수치"로 설정합니다.

Q439: 계수 검증을 수행한 후 어떤 결과를 확인할 수 있나요?  
A 계수 검증을 수행한 후에는 분석 결과로 숫자 데이터의 비교 결과를 확인할 수 있습니다. 특히, 파란색 영역에 표시된 값은 정상으로 판단되며, 그 외의 값들은 오류로 판단된 값입니다. 이를 통해 데이터의 품질을 평가하고, 필요한 조치를 취할 수 있습니다. 분석 결과는 화면에 표시되며, 각 항목의 정확성과 일관성을 검토할 수 있습니다.

Q440: 오류로 판단된 값은 어떻게 처리해야 하나요?  
A 오류로 판단된 값은 데이터 품질에 문제가 있을 수 있으므로, 해당 값을 검토하고 수정해야 합니다. 먼저, 오류가 발생한 데이터를 확인하고, 그 원인을 분석합니다. 이후, 필요에 따라 데이터를 수정하거나, 관련 부서와 협의하여 문제를 해결합니다. 수정 후에는 다시 계수 검증을 수행하여 오류가 해결되었는지 확인하는 것이 중요합니다.

Q441: 분석 항목을 추가하려면 어떤 절차를 따라야 하나요?  
A 분석 항목을 추가하기 위해서는 다음 단계를 따라야 합니다. 1) 분석 설정 화면에서 '분석항목' 섹션으로 이동합니다. 2) [추가] 버튼을 클릭합니다. 3) 분석 항목 정보를 입력합니다. 예를 들어, '분석 항목 명'에는 '운영계 코드 테이블 데이터 수'를 입력하고, '대상 테이블'에는 'C##DQ_TARGET.TB_CD'를 입력합니다. 4) '대상 컬럼-비교기준'에는 'TB_CD.CD_GRP_ID', '대상 컬럼-비교대상'에는 'COUNT(CD_ID)'를 입력합니다. 5) '조회 SQL'에는 'SELECT CD_GRP_ID, COUNT(CD_ID) FROM C##DQ_TARGET.TB_CD GROUP BY CD_GRP_ID'를 입력합니다. 6) 입력이 완료되면 [저장] 버튼을 클릭하여 분석 항목을 저장합니다. 

Q442: 대상 컬럼을 자동으로 입력하려면 어떻게 해야 하나요?  
A 대상 컬럼을 자동으로 입력하려면, 먼저 '조회 SQL'을 입력한 후 [대상 컬럼 동기화] 버튼을 클릭해야 합니다. 이 버튼을 클릭하면 시스템이 입력한 SQL을 분석하여 대상 컬럼을 자동으로 동기화합니다. 

Q443: 운영계와 개발계의 분석 항목을 어떻게 구분하나요?  
A 운영계와 개발계의 분석 항목은 각각 다른 테이블을 참조하여 구분합니다. 운영계의 경우 'C##DQ_TARGET.TB_CD' 테이블을 사용하고, 개발계의 경우 'C##DQ_TARGET_DEV.TB_CD' 테이블을 사용합니다. 각 항목의 '분석 항목 명'에 명확히 '운영계' 또는 '개발계'를 기재하여 구분할 수 있습니다. 

Q444: 분석 항목을 비교 기준으로 설정하려면 어떻게 해야 하나요?  
A 분석 항목을 비교 기준으로 설정하려면, 먼저 분석 항목 목록에서 운영계 코드 테이블 데이터 수 분석 항목을 찾아 체크박스를 선택합니다. 이렇게 하면 해당 분석 항목이 비교 기준으로 설정됩니다. 

Q445: 분석 항목 정보를 입력할 때 주의할 점은 무엇인가요?  
A 분석 항목 정보를 입력할 때는 각 필드에 정확한 정보를 입력하는 것이 중요합니다. '대상 테이블', '대상 컬럼-비교기준', '대상 컬럼-비교대상' 등의 정보를 잘못 입력하면 분석 결과에 오류가 발생할 수 있습니다. 따라서 입력 후 다시 한 번 확인하는 것이 좋습니다.

Q453: 다중 갭 분석을 실행하려면 어떻게 해야 하나요?  
A: 먼저, 화면에서 [저장] 버튼을 클릭한 후, [분석 실행] 버튼을 클릭합니다. 그 다음, ‘진단 결과’ 메뉴로 이동하여 ‘다중 갭 분석’을 선택합니다. 이후 ‘업무’ 항목에서 ‘운영-데이터품질’을 선택하고 [조회] 버튼을 클릭하여 결과를 확인할 수 있습니다.

Q454: 데이터 검증을 수행하려면 어떤 절차를 따라야 하나요?  
A: 데이터 검증을 수행하기 위해서는 먼저 다중 갭 분석을 실행한 후, ‘진단 결과’ 메뉴에서 관련된 결과를 조회해야 합니다. 그런 다음, ‘계수 검증의 진단결과’를 클릭하여 상세 결과를 확인하고, 하단에서 “운영계와 개발계의 코드테이블의 코드는 그룹별로 동일”이라는 라인의 우측에 있는 [진단 상세 결과] 버튼을 클릭합니다. 이 과정을 통해 코드 그룹 아이디를 기준으로 비교 결과를 조회할 수 있습니다.

Q455: 데이터 검증을 설정하려면 어떤 메뉴를 선택해야 하나요?  
A: 데이터 검증을 설정하기 위해서는 웹 화면에서 '진단 실행' 메뉴를 클릭한 후, 그 아래의 '다중 갭 분석' 메뉴를 선택하세요.

Q456: 다중 갭 분석 정보를 입력할 때 어떤 버튼을 클릭해야 하나요?  
A: 다중 갭 분석 정보를 입력하기 위해서는 '추가' 버튼을 클릭한 후, 필요한 정보를 입력하고 '저장' 버튼을 클릭하여 저장하세요.

Q457: 다중 갭 분석을 설정할 때 추천하는 항목 예시는 무엇인가요?  
A: 추천하는 항목 예시는 다음과 같습니다:  
- 다중 갭 분석 명: 운영계와 개발계의 코드그룹 데이터는 동일해야함.  
- 대상 업무: 운영-데이터품질  
- 상태: 사용중  
- 분석 유형: 데이터검증  
- 분석 설정 - 비교 기준: 전체비교  
- 분석 설정 - 동일데이터제외: 아니오  

Q458: 분석항목을 추가하려면 어떤 절차를 따라야 하나요?  
A: 분석항목을 추가하기 위해서는 분석항목에서 '추가' 버튼을 클릭한 후, 필요한 정보를 입력하고 '저장' 버튼을 클릭하세요.

Q459: 대상 컬럼을 자동으로 입력하려면 어떻게 해야 하나요?  
A: 대상 컬럼을 자동으로 입력하려면 먼저 조회 SQL을 입력한 후, '대상 컬럼 동기화' 버튼을 클릭하면 자동으로 입력됩니다.

Q460: 분석항목 정보를 입력할 때 어떤 SQL을 사용해야 하나요?  
A: 분석항목 정보를 입력할 때 사용할 SQL 예시는 다음과 같습니다:  
```sql
SELECT CD_GRP_ID, 
       CD_GRP_NM, 
       CD_GRP_DESC 
FROM   C##DQ_TARGET.TB_CD_GRP
```

Q461: 분석항목을 추가하는 과정에서 몇 번의 '추가' 버튼 클릭이 필요한가요?  
A: 분석항목을 추가하는 과정에서 '추가' 버튼을 두 번 클릭해야 합니다. 첫 번째는 다중 갭 분석 정보를 입력할 때, 두 번째는 분석항목 정보를 입력할 때입니다.

Q466: 대상 컬럼을 자동으로 입력하려면 어떻게 해야 하나요?  
A: 조회 SQL을 입력한 후, 화면에서 [대상 컬럼 동기화] 버튼을 클릭하면 자동으로 대상 컬럼이 입력됩니다. 이 버튼은 SQL 입력란 바로 아래에 위치해 있습니다.

Q467: 분석 항목을 설정하는 방법은 무엇인가요?  
A: 분석 항목을 설정하려면, 먼저 '운영계 코드 그룹' 분석 항목을 선택하고 체크합니다. 이후 [저장] 버튼을 클릭하여 설정을 저장합니다.

Q468: 분석을 실행하려면 어떤 절차를 따라야 하나요?  
A: 분석을 실행하려면 먼저 [저장] 버튼을 클릭한 후, [분석 실행] 버튼을 클릭합니다. 이 버튼은 저장 후 화면의 상단에 위치해 있습니다.

Q469: 진단 결과를 확인하려면 어떻게 해야 하나요?  
A: 진단 결과를 확인하려면 ‘진단 결과’ 메뉴로 이동한 후, ‘다중 갭 분석’ 메뉴를 클릭합니다. 이후 ‘업무’에서 ‘운영-데이터품질’을 선택하고 [조회] 버튼을 클릭합니다.

Q470: 데이터 검증의 진단 결과를 어떻게 확인하나요?  
A: 데이터 검증의 진단 결과를 확인하려면, 진단 결과 화면에서 해당 결과를 클릭한 후, 하단에 있는 “운영계와 개발계의 코드그룹 데이터는 동일해야함.” 라인의 우측에 있는 [진단 상세 결과] 버튼을 클릭합니다.

Q473: 코드 그룹 아이디를 기준으로 비교 결과를 조회하려면 어떻게 해야 하나요?  
A: 비교 결과를 조회하기 위해서는 먼저 웹 페이지에 로그인한 후, 상단 메뉴에서 '데이터 분석'을 클릭합니다. 그 다음 '비교 결과 조회' 옵션을 선택하세요. 이후 '코드 그룹 아이디' 입력란에 원하는 코드 그룹 아이디를 입력하고 '조회' 버튼을 클릭하면 해당 코드 그룹 아이디에 대한 비교 결과가 표시됩니다.

Q474: 코드 그룹 아이디가 무엇인지 어떻게 확인할 수 있나요?  
A: 코드 그룹 아이디는 일반적으로 데이터베이스나 관리 시스템의 '코드 관리' 섹션에서 확인할 수 있습니다. 웹 페이지에 로그인한 후, 상단 메뉴에서 '코드 관리'를 클릭하고, 목록에서 원하는 코드 그룹을 찾아 해당 항목을 클릭하면 코드 그룹 아이디가 상세 정보로 표시됩니다.

Q475: 비교 결과가 조회되지 않는 경우 어떻게 해야 하나요?  
A: 비교 결과가 조회되지 않는 경우, 입력한 코드 그룹 아이디가 정확한지 다시 확인하세요. 또한, 시스템에 문제가 있을 수 있으니, 상단 메뉴에서 '시스템 상태'를 확인하여 정상 작동 중인지 확인합니다. 만약 문제가 지속된다면, 고객 지원팀에 문의하여 도움을 요청하세요.

Q476: 여러 개의 코드 그룹 아이디를 한 번에 조회할 수 있나요?  
A: 네, 여러 개의 코드 그룹 아이디를 한 번에 조회할 수 있습니다. '비교 결과 조회' 페이지에서 '코드 그룹 아이디' 입력란에 각 아이디를 쉼표로 구분하여 입력한 후 '조회' 버튼을 클릭하면 해당 코드 그룹 아이디들에 대한 비교 결과가 동시에 표시됩니다.

Q477: 조회된 비교 결과는 어떻게 저장하나요?  
A: 조회된 비교 결과를 저장하려면, 결과 화면에서 '내보내기' 버튼을 클릭합니다. 이후 파일 형식(예: CSV, Excel 등)을 선택하고 '저장' 버튼을 클릭하면 해당 파일이 다운로드됩니다. 다운로드한 파일은 컴퓨터에서 열어 확인할 수 있습니다.
